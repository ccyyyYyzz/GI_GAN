# HQ-05 — RestoreFormer cross-attention fusion: content-adaptive `β(x)` in the null space

**Status:** proposed (paper-ready method subsection + proposition + experiment design; no results — the ablation must be run).

**One-line.** Replace the image-independent scalar `B` / radial band schedule with a *content-adaptive* per-pixel blend field `β(x)` produced by a lightweight cross-attention head (queries from the anchor/structure features, keys/values from the VQGAN HQ code embeddings), fused strictly inside the exact null projector so `A x_hat = y` holds for *any* `β`; `β ≡ B` is a special case, so at worst it matches the tuned scalar baseline.

**Card-numbering note.** The deck (`HQ_GI_INNOVATION_POINTS.md`) cites RestoreFormer as R059 and CodeFormer as R058. In the library file `GI99_reading_cards.json` the correct file tags are **R060 = RestoreFormer** (Wang et al., CVPR 2022), **R059 = CodeFormer** (Zhou et al., NeurIPS 2022), **R058 = VQFR** (Gu et al., ECCV 2022), **R055 = VQ-VAE-2**, **R056 = VQGAN**, **R057 = ViT-VQGAN**. Citations below use the library (R06x) tags.

## 1. Motivation and the gap this closes

Our locked detail fusion writes null-space content by a two-residual blend
`x_hat_B = x0 + P0( d_A + W·(d_G − d_A) )`, where `d_A = P0(x_A − x0)` (VQAE structure residual), `d_G = P0(x_G − x0)` (VQGAN detail residual), and `W` is either a **scalar** `B` or a **radial frequency band schedule** `weight_map(w, radial_masks)` (`vqgan_detail_fusion.py:290–302`). Both choices are **image-independent**: the same mixing law is applied to every image and every spatial location.

This is exactly the fusion primitive that the blind-face-restoration line already identified as suboptimal. RestoreFormer (R060) frames restoration as fusing *"two sources of information: the degraded face with identity information and the priors with high-quality details"* and shows that a **fully-spatial multi-head cross-attention** over an undegraded HQ VQ dictionary beats pixel-wise Spatial-Feature-Transform (SFT) fusion *because SFT operates pixel-wise and ignores image context* — smooth regions want structure, textured regions want detail, and a fixed spatial law cannot tell them apart. CodeFormer (R059) makes the same point at the level of a single scalar knob: its Controllable Feature Transformation gates input features by one scalar `w ∈ [0,1]`, a global fidelity–quality trade identical in *interface* to our `B`. VQ-VAE-2 (R055) supplies the structural reason the two residuals are separable in the first place — a *global-structure* code (top) and a *local-detail* code (bottom) encode complementary information — which is exactly what `d_A` (structure) and `d_G` (detail) are.

The gap: **which prior is trustworthy is content- and region-dependent, but our mixing weight is not.** HQ-05 makes the weight a learned function of the image, `β(x)`, while keeping every property of the exact-consistency invariant.

## 2. Formulation

Let `x0` be the LMMSE anchor (`hq.EmpiricalLMMSE.anchor`), `x_A, x_G` the frozen multi-seed VQAE/VQGAN refiner outputs (`vqgan_detail_fusion.py:128–134`), and

```
d_A = P0( flatten(x_A − x0) ),   d_G = P0( flatten(x_G − x0) )
```

the null-projected residuals already produced by `prep_residuals` (`vqgan_detail_fusion.py:332–337`). Define a **per-pixel blend field** `β(x) ∈ [0,1]^{H×W}` and fuse

```
x_hat_β = x0 + P0( d_A + β(x) ⊙ (d_G − d_A) ),          (HQ-05.1)
```

where `⊙` is elementwise (broadcast over channels) multiplication in image space, then flattened. Because `P0` is applied *after* the elementwise product and `A P0 = 0` exactly (`src/projections.py`), we have

**Proposition 1 (unconditional exact consistency).** *For every field `β : ℝ^{H×W} → ℝ^{H×W}` (learned, bounded, or otherwise), the reconstruction (HQ-05.1) satisfies `A x_hat_β = y` exactly (to the projector's numerical floor, ~2e-15), and `audit_flat(·, y)` is a no-op up to float drift.*

*Proof.* `x_hat_β = x0 + P0(u)` with `u = d_A + β⊙(d_G − d_A)`, a fixed flat vector once `β` is evaluated. `A x_hat_β = A x0 + A P0 u = A x0 = y` since `A x0 = y` by the data-anchor property of the LMMSE anchor and `A P0 = 0`. The value of `β` never enters `A x_hat_β`. ∎

**Corollary (baseline is reachable).** Setting `β ≡ B` (a constant field) recovers the current scalar arm exactly; setting `β = weight_map(w, masks)` broadcast to a spatial field recovers the radial band arm (the radial law is a frequency-domain operator, so band-equivalence is exact only for the frequency-band parameterization of §3.2, otherwise `β` is its image-space analogue). Hence the learned fusion's reachable set *strictly contains* the tuned baseline, and the trained optimum can only match-or-beat it on the training objective.

This is the crux of the honesty of the point: **quality is bought only inside `P0`**, and the invariant is preserved *by construction*, not by a penalty. This is precisely the "exact-decoupling answer" that VQFR (R058) and RestoreFormer (R060) approach only heuristically — VQFR needs a parallel decoder + texture-warping module and a compression-patch-size knob to stop input-fidelity from "contaminating" invented detail; here contamination in the measured coordinates is impossible because detail lives in `Null(A)`.

## 3. Architecture — the `β(x)` head

Two variants; both are lightweight and trained **only** to reduce LPIPS at fixed (automatic) consistency, with the heavy priors and the projector frozen.

### 3.1 Variant A — cross-attention head (the RestoreFormer-grounded design)

Following RestoreFormer's fully-spatial cross-attention (R060), but repurposed to emit a *scalar blend per location* rather than to synthesize features:

- **Queries `Q`.** A 2-layer conv stem over the fidelity/structure stream `(x0, x_A, d_A)` → feature map `F_q ∈ ℝ^{H×W×C}` (C≈64). These carry "what the measurement + structure prior already support here."
- **Keys/Values `K,V`.** The VQGAN HQ code embeddings for this image: take `z_G = VQGAN.encode(x_G)`, quantize, and read the codebook embeddings `E(z_G) ∈ ℝ^{h×w×D}` (the same `decode_embeddings` codebook already wired at `vqgan_detail_fusion.py:130–134`). These are the "undegraded HQ dictionary elements" of RestoreFormer, i.e. what the detail prior *offers* here.
- **Cross-attention.** Multi-head attention `Attn(Q, K, V)` with `Q` from `F_q` and `K,V` from `E(z_G)` (bilinearly matched to the `H×W` grid). Output `F_att ∈ ℝ^{H×W×C}`.
- **Blend head.** `β(x) = σ( Conv1x1( [F_att, F_q, |d_G − d_A|] ) )`, a sigmoid to bound `β ∈ [0,1]`. The disagreement magnitude `|d_G − d_A|` is fed in as an explicit prior-conflict cue (this is the same quantity HQ-08 uses closed-form; here it is one input feature among several).

Parameter budget ≈ 60–150k (2-layer stem + one attention block + 1×1 head), matched to the 64×64 STL10 testbed. Trainable params: only this head.

### 3.2 Variant B — band-resolved `β` (cheaper, stays closest to the current radial fusion)

Emit a small vector `β_band ∈ [0,1]^{K}` per image (K = `N_BANDS`) from a global-pooled version of the stem, and apply it through the *existing* `apply_band`/`radial_masks` machinery: `weight_map(β_band, masks)`. This is a **per-image, frequency-resolved** generalization of the current radial schedule (which uses a fixed `w`), strictly less expressive than Variant A but a direct, low-risk drop-in that reuses `apply_band` verbatim (`vqgan_detail_fusion.py:283–299`). Recommended as the *first* run because it isolates "content-adaptivity of the band schedule" from "per-pixel spatial adaptivity."

### 3.3 Where it plugs in

Add a spec kind `("beta", head)` to `fuse` (`vqgan_detail_fusion.py:290`). The only change to the fusion body:

```python
if kind == "beta":
    beta = head(x0_img, x_A_img, d_A_img, d_G_img)      # H×W in [0,1], no grad through P0 needed for fwd
    delta_img = measurement.unflatten_img(d_G - d_A)
    dd = measurement.flatten_img(beta * delta_img)
d_F = projector.null_project_flat(d_A + dd)             # unchanged
xhat = projector.audit_flat(x0f + d_F, y)               # unchanged
```

Everything downstream (`audit_flat`, `fast_means`, `select_operating_point`, the val→dev gate) is untouched. The head is the only new trainable object; `x0`, `x_A`, `x_G`, `d_A`, `d_G`, `P0` are all frozen/cached.

## 4. Training recipe

- **Data.** STL10 64×64 train split, same operators (structured DCT+Hadamard+random) at 2/5/10%. Cache `x0, x_A, x_G, d_A, d_G` once per image (they are frozen) so training only forward-passes the head + the differentiable projector.
- **Objective.** `L = LPIPS(clamp(x_hat_β), truth) + γ·L_dist`, where `L_dist` is an RMSE (or Charbonnier) term with a *small* weight `γ` chosen so that the PSNR of the selected point stays within the `balanced` gate's `−0.5 dB` band (mirrors `select_operating_point`'s balanced constraint at `vqgan_detail_fusion.py:344`). No consistency term is needed — Proposition 1 makes it free.
- **Differentiability.** `null_project_flat` is a linear operator, hence differentiable; gradients flow to the head through `P0` cleanly. Clamp is applied only in the metric, straight-through if needed.
- **Regularization toward the baseline (critical, given the honest risk in §7).** Initialize / warm-start so that at step 0 the head emits a constant `β ≡ B*` (the locked scalar). Concretely: bias the sigmoid pre-activation to `logit(B*)` and zero-init the last 1×1 conv weights. Then stage-0 reproduces the current Pareto point exactly (Corollary), and training can only depart from it if the departure lowers the objective. This directly guards against the failure mode where a learned gate collapses back to ≈scalar and buys nothing (see §7).
- **Optional CodeFormer-style second knob.** Expose a global `w ∈ [0,1]` that scales the head's *deviation* from the constant baseline: `β_w = B* + w·(β(x) − B*)`. `w=0` → tuned scalar, `w=1` → full content-adaptive. This gives a governed, monotone "how much spatial adaptivity" dial orthogonal to the perception–distortion `B`-axis, mirroring CodeFormer's `w` (R059) but with the categorical difference that here *every* `w` keeps `A x_hat = y` exact (R059's `w` moves the reconstruction off the measurement).
- **Cost.** Head is tiny; residuals cached; ≈1 GPU-hour per operating point, comparable to HQ-06.

## 5. Inference recipe

1. Load frozen priors, refiners, LMMSE anchor, exact projector, and the trained `β`-head (checkpoint slots alongside `refiner_ckpt`).
2. For each test image: compute `x0`, `x_A`, `x_G`, `d_A`, `d_G` (cached path in `regen_seed`), evaluate `β(x)`, fuse via (HQ-05.1), finish with one `audit_flat` against float drift.
3. Select the single reported operating point on the **locked val split**, freeze, score on **dev** through the existing `cmd_gate` — identical firewall to every other arm. Verify `relmeaserr ≤ 1e-5` (it will hold by Proposition 1).

## 6. Ablation / experiment design

**Primary comparison (the claim).** At 5% seed0, through `cmd_gate`:
- Arm 0: `fusion_balanced` at the locked scalar `B*` (the `−32.6% LPIPS` operating point) — baseline.
- Arm 1: **HQ-05 Variant B** (per-image band `β_band`).
- Arm 2: **HQ-05 Variant A** (per-pixel cross-attention `β(x)`).
- Report LPIPS and PSNR (and RAPSD) for each. **Success = Arm 2 (and/or Arm 1) LPIPS strictly below the scalar baseline at equal-or-smaller PSNR cost, and it holds on seeds 1 and 2** (multi-seed gate).

**Consistency check (the invariant).** For all arms, confirm `relmeaserr ≤ 1e-5`; plot the `β`-swept flat residual vs. CodeFormer's `w`-swept residual drift as the categorical-difference figure (R059 contrast).

**Adaptivity attribution (does content-adaptivity actually help?).**
- (a) *Collapse test.* Compare Arm 2 against Arm 0 *and* against the head with the §4 baseline warm-start but **0 training steps** (≡ scalar). If trained Arm 2 ≈ collapse baseline, the head learned nothing beyond the scalar — report this honestly.
- (b) *Spatial-variance probe.* Measure `std_spatial(β(x))` per image and correlate with per-image LPIPS gain. A gain that *scales with* spatial variance is the positive evidence that region-adaptivity (not just a re-tuned global scalar) is the source.
- (c) *Oracle ceiling.* Grid a per-band `β` on val (closed-form, no training) to bound how much a *band-resolved* schedule can possibly buy; this upper-bounds Variant B and contextualizes Variant A's extra spatial degrees of freedom.

**Rate sweep.** Repeat the primary comparison at 2% and 10%. Expectation: the content-adaptive advantage is largest where the two priors *disagree most*, which is the lowest rate (2%), because that is where the "trust structure vs. trust detail" decision varies most across regions.

**Cross-check against HQ-06 (MoDL).** Both are spatially-adaptive learned refiners of the *same two residuals*; run both under the identical gate so the paper can state which of "reweight-only" (HQ-05) vs "iterative-synthesize" (HQ-06) wins — the deck's own open question (§ "marginal-gain risks").

## 7. Honest assessment of expected gain — **likely marginal, not large**

I flag this plainly, consistent with the deck's own risk note (`HQ_GI_INNOVATION_POINTS.md` marginal-gain risks: *"HQ-5 (β(x)) vs HQ-6 (MoDL) — both are spatially-adaptive learned blends of the same residuals; a prior learned-gate attempt already underperformed. Real risk that a spatial gate re-learns ≈ the scalar and buys little"*):

- **Bounded by the reachable set.** `β(x)` only *reweights* the two fixed residuals `d_A, d_G`; it cannot synthesize content that is absent from both. Its ceiling is the best pointwise convex combination of `{d_A, d_G}` in the null space. Where `d_A` and `d_G` agree, any `β` is equivalent; the head can only help where they *disagree region-to-region within an image*. On 64×64 STL10 the two frozen refiners may not disagree enough, spatially, to leave much on the table.
- **Prior negative evidence.** A previous *learned-gate* attempt in this program underperformed, and the classic failure is that the gate collapses back to ≈ the tuned scalar. The §4 baseline warm-start makes "no worse than scalar" essentially guaranteed, but "meaningfully better" is not.
- **Where it could still be worth it.** (i) The *governance/story* value is real regardless of magnitude: it converts the two-ledger framing of RestoreFormer/VQFR (R060/R058) into a fusion that is *exact* rather than heuristic, and gives a clean head-to-head figure against CodeFormer's `w` (R059). (ii) If any lift appears, it should concentrate at 2% (max prior disagreement), which is the regime the paper most wants to strengthen. (iii) Variant B is cheap enough that the oracle band-grid (6c) can *predict the ceiling before training*, so the head is only trained if the ceiling is worth it.

**Recommendation.** Run the closed-form oracle band-grid (§6c) and the spatial-variance probe (§6b) *first*; only train the full cross-attention head if the oracle shows a non-trivial band-resolved ceiling and per-image `β` variation correlates with LPIPS gain. Treat HQ-05 as a **higher-ceiling companion to HQ-08** (per-image closed-form `B`) and a **lower-ceiling sibling of HQ-06** (MoDL, which can synthesize, not just reweight): if a bigger quality jump is the goal, HQ-06 is the better bet; if a principled *exact* answer to the RestoreFormer/VQFR fusion problem is the goal, HQ-05 is the cleaner contribution.

## 8. Grounding (library card tags)

- **R060 RestoreFormer** — fully-spatial multi-head cross-attention, queries from corrupted input, keys/values from an undegraded HQ VQ dictionary; beats pixel-wise SFT by using image context. *Direct architectural template for the `β(x)` head (§3.1) and the two-ledger fusion framing (§1).*
- **R059 CodeFormer** — single scalar `w ∈ [0,1]` (Controllable Feature Transformation) trading fidelity vs quality on *entangled* features. *Grounds the optional §4 governed `w` knob and the categorical "our knob costs nothing on the measurement" contrast (§4, §6).*
- **R058 VQFR** — parallel decoder + texture-warping to stop input fidelity from contaminating invented codebook detail; heuristic. *Grounds §2's claim that `P0` is the exact-decoupling answer to VQFR's contamination problem.*
- **R055 VQ-VAE-2** — explicit factorization into a global-structure code (top) and local-detail code (bottom). *Grounds why `d_A` (structure) and `d_G` (detail) are separable, hence why a content-adaptive per-region choice between them is meaningful (§1, §2).*
- **R056 VQGAN** — perceptual+adversarial codebook autoencoder that synthesizes texture; its adaptive `λ` balances fidelity vs realism at the decoder's last layer. *Source of `d_G`; the frozen codebook embeddings are the keys/values in §3.1; contrast: its `λ` is a soft whole-image trade, our `β` is exact and null-confined.*
- **R057 ViT-VQGAN** — factorized / L2-normalized codebook (lookup subspace decoupled from embedding). *Background for the code-embedding key/value construction; supports high-fidelity detail prior.*

*Invariant respected: all quality is bought inside `P0`; `A x_hat_β = y` to ~2e-15 for every `β` (Proposition 1). No numbers are invented — the only quantitative reference (`−32.6% LPIPS`) is the existing locked baseline, and every performance claim is stated as a target for the ablation to test.*
