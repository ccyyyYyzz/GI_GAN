# HQ-13 — Fusion-in-the-Loop Operator Learning (co-design the sensing operator `A_θ` and the fusion dial `B`)

**Status:** proposed (paper-ready method subsection + proposition + training/inference recipe + ablation design). Not yet run; the differentiable infrastructure exists but the exact projector must be made differentiable in `A_θ` (see §7 gaps).

**One-line.** Stop choosing the ghost-imaging operator blind to the reconstructor: make the sensing rows a learnable `A_θ` and back-propagate a perceptual reconstruction loss *through the exact range–null fusion* into `A_θ` and the fusion dial `B` jointly, so the measured rows capture exactly the content our VQAE+VQGAN fusion cannot invent well — while `A_θ P_0(A_θ)=0` (exact consistency) is held analytically at every step and never enters the loss.

**Invariant respected.** Exact consistency is preserved: for the *learned* operator, reconstruction remains `x̂ = x0(A_θ) + P_0(A_θ)(d_A + B(d_G − d_A))` with `A_θ x̂ = y` to numerical zero by construction of the projector built from `A_θ`. Quality is still bought only inside the null space; the novelty is that we *shape the null space itself* by choosing `A_θ` for the fusion, rather than filling a fixed null space better (HQ-1…HQ-11) or filling it with a perceptually-chosen-but-reconstructor-agnostic operator (HQ-12).

---

## 1. Motivation and the gap over HQ-12

Every current operator (DCT / Hadamard / random rows at 2/5/10 %) and even the perception-weighted operator of HQ-12 is fixed **before** the priors act. HQ-12 chooses the `m` measured rows to span perceptually dominant directions using an LPIPS-Jacobian Gram `G = E[JᵀJ]` — but it is *blind to what our specific VQAE(structure)+VQGAN(detail) fusion can already hallucinate for free*. If the fusion reliably reconstructs, say, low-frequency structure and smooth texture from a partial measurement, spending measured rows there is wasted: those directions are perceptually cheap to invent. HQ-13 closes exactly this loop: it measures the content the fusion gets *wrong*, i.e. the null directions where the fusion's expected perceptual error is largest, which is a strictly reconstructor-conditioned criterion that HQ-12's operator-only Gram cannot see.

This is the ghost-imaging / single-pixel-imaging instance of **joint pattern–reconstruction co-design**: SPI reviews (card #98, Song et al. 2025) explicitly note that DL-SPI reconstruction quality "depends heavily on" the modulation matrix `H`, and learned-operator inverse-problem work (Learned Primal-Dual, card #30, Adler & Öktem 2018; Variational Network, card #29, Hammernik et al. 2018) demonstrates that embedding the forward operator as a differentiable layer and training end-to-end recovers content that operator-agnostic pipelines miss (Learned Primal-Dual's Fig. 4: only the end-to-end method recovers the true small feature; the others render it "indistinguishable from false features of the same size and contrast"). HQ-13 keeps the *priors frozen* and trains only the two cheapest, most physically meaningful degrees of freedom — the sensing rows and the fusion dial.

## 2. Method subsection (paper-ready)

### 2.1 Setup and notation

Let `x ∈ R^n` be a 64×64 STL10 image (`n = 4096`). The ghost-imaging / single-pixel forward model is `y = A_θ x + η` with `A_θ ∈ R^{m×n}`, `m = ⌊ρ n⌋` at sampling ratio `ρ ∈ {0.02, 0.05, 0.10}` (card #98's `B = H T + η`; here `η = 0` on the clean testbed, matching the locked line). The rows of `A_θ` are produced by a **learnable pattern bank** `Θ ↦ A_θ`:

- continuous logits `Θ ∈ R^{m×n}` → soft patterns `P_soft = σ(Θ/τ)` (transmission-constrained), → physical binary patterns `P = STE(P_soft)` via straight-through binarization (forward = hard `1[P_soft>½]`, backward = soft gradient), → the **effective centered/standardized operator** `A_θ = center_standardize(P)` used in the measurement (mirrors `LearnablePatternBank.get_effective_A` with `effective_A_mode="centered_standardized"`). Binarization and transmission constraints make `A_θ` DMD-realizable; `τ` anneals hard→soft over training.

Given `A_θ`, build the exact projector `Π_θ := ExactRangeNullProjector(A_θ)` (matrix-free; §7). It exposes:
- range/data anchor `x0_θ(y)`  — we use the **measurement-audited LMMSE anchor** `x0 = μ + Σ Aᵀ(A Σ Aᵀ + σ²I)⁻¹(y − Aμ)` re-fitted for `A_θ` (`hq.EmpiricalLMMSE`), which reduces to the range back-projection `A_θ⁺ y` in the noiseless limit;
- null projector `P_0^θ = I − A_θ⁺ A_θ`, with the **hard identity** `A_θ P_0^θ = 0`.

### 2.2 The frozen fusion head

The VQAE structure residual and VQGAN detail residual are computed and null-projected exactly as in `prep_residuals`:
```
d_A = P_0^θ ( x_A − x0 ),   d_G = P_0^θ ( x_G − x0 ),
```
where `x_A = VQAE(x0)` and `x_G = VQGAN(x0)` are produced by the **frozen** multi-seed refiners (`refiner_ckpt`; decode via `decode_embeddings`). The fusion is the locked scalar blend
```
x̂_B = x0 + P_0^θ ( d_A + B ( d_G − d_A ) ) = x0 + d_A + B ( d_G − d_A ),
```
(the second equality holds because `d_A, d_G` already live in `Null(A_θ)`; `P_0^θ` is idempotent). This is exactly `vqgan_detail_fusion.fuse` with a scalar `B`, and it is differentiable in `B` and — through `x0`, `d_A`, `d_G`, `P_0^θ` — in `Θ`.

### 2.3 The training objective

Only `Θ` and `B` (optionally a per-band vector `B` reusing `radial_masks`, keeping HQ-13 a strict generalization of the current dial) are trainable. Priors are frozen. Minimize a perception-weighted distortion loss over STL10-train:
```
L(Θ, B) = E_x [ LPIPS( x̂_B(x), x ) + γ · ‖ x̂_B(x) − x ‖²₂ ] + λ_P · R_pattern(Θ)
```
- `LPIPS` drives the perceptual ceiling (card #52); `γ‖·‖²` is a light distortion anchor so the operator does not trade away PSNR (the paper reports at a **bounded PSNR** budget — see §5).
- `R_pattern(Θ)` = the existing `total_pattern_loss` (energy, decorrelation, binary-ness, second-order RIP surrogate, contrast) that keeps `A_θ` physically realizable and well-conditioned (`pattern_regularizers.total_pattern_loss`). This is the only thing preventing a degenerate operator; consistency is *not* in the loss.
- Crucially, **there is no `‖A_θ x̂ − y‖` term**: consistency is analytic (`A_θ P_0^θ = 0`), so the loss is free to spend all capacity on perceptual quality. This is the structural advantage over Learned Primal-Dual / Deep Decomposition (card #16), which must either learn or CG-solve data consistency; we own it exactly.

### 2.4 Straight-through gradient path

Gradients flow: `L → x̂_B → {B, d_A, d_G, x0, P_0^θ} → Π_θ → A_θ → P = STE(P_soft) → P_soft = σ(Θ/τ) → Θ`. The STE step passes the soft Jacobian on the backward pass. Gradient flow through the pattern-bank → measurement → reconstruction → L1 chain is already verified end-to-end in `sanity_learnable_patterns.py` (`gradient_check`: `logits.grad` finite, norm > 0); HQ-13 only extends the *head* of that chain from a UNet reconstructor to the frozen exact fusion, and requires the projector's `A_θ`-dependence to be differentiable (§7).

## 3. Proposition (reachability / no-regret vs the fixed operator)

**Proposition 1 (co-design weakly dominates the fixed-operator fusion at equal budget).**
Let `A₀` be any fixed operator (DCT/Hadamard/random or the HQ-12 perceptual-PCA operator) with its tuned dial `B₀`, and let `x̂^{A₀,B₀}` be its exact fusion, achieving expected loss `L₀ = E[LPIPS + γ‖·‖²]`. Consider the co-design family `F = { (A_θ, B) : A_θ ∈ 𝒜, B ∈ [0,1] }` where `𝒜` is the set of transmission-/binary-feasible learnable operators. If `A₀ ∈ 𝒜` (i.e. the fixed operator is representable by the pattern bank up to the standardization map — true for random/Hadamard inits, which are the pattern-bank's default `pattern_init`), then
```
min_{(A_θ,B) ∈ F}  L(A_θ, B)  ≤  L(A₀, B₀),
```
with the minimizer's consistency `‖A_θ x̂ − y‖ / ‖y‖ = O(ε_float)` unchanged. **Proof sketch.** `(A₀, B₀) ∈ F` by assumption, so the minimum over `F` is no larger than its value at that point; `L` never includes a consistency term because `A_θ P_0^θ = 0` holds for every `A_θ ∈ 𝒜` (projector construction), so feasibility is preserved throughout the search. ∎

This is the operator-space analogue of the HQ-9/HQ-10 "`B₁=1` / endpoint-α recovers the baseline, so it can only match-or-beat" argument, lifted from the dial to the sensing matrix. It is a **weak** dominance (no-regret in expectation over the training distribution); it does **not** promise a large gap, and standardization can make `A₀ ∈ 𝒜` only approximate (§6).

**Corollary (where the gain must come from).** Because the fusion head is frozen and the anchor `x0` is the (near-)MMSE range estimate, `L` decomposes as an operator-independent range term plus a null-space perceptual-error term. Co-design can only reduce the second: it moves perceptually-expensive directions out of `Null(A_θ)` and into the measured row space. The realizable gain is therefore bounded by how much of the fusion's residual perceptual error lives in directions the operator can *choose to measure* under the `m`-row and binary constraints — see §6 for why this is likely modest over HQ-12.

## 5. Training / inference recipe

**Training (per sampling ratio `ρ`, per seed on train split):**
1. Init `Θ` from a strong fixed operator (`pattern_init="hadamard"` or `"rademacher"`) so stage-0 ≈ the current baseline (Proposition 1 anchor). Init `B = B*` (the locked scalar).
2. Fit `EmpiricalLMMSE` statistics for the initial `A_θ`; **re-fit `x0` statistics periodically** (every `N_refit` steps) as `A_θ` drifts — or, cheaper, keep the LMMSE in its noiseless limit `x0 = A_θ⁺ y` computed through the differentiable projector so it tracks `A_θ` automatically.
3. For a few hundred steps: sample a batch, `y = A_θ x`; compute `x0, d_A, d_G` (VQAE/VQGAN frozen, one forward each), `x̂_B`; loss §2.3; backprop into `Θ, B`; anneal `τ`.
4. Every step, assert `audit_flat(x̂_B, A_θ, y)` → `relmeaserr ≤ 1e-5` (a **hard invariant check**, not a loss).
5. Freeze `Θ, B`; binarize `A_θ` at `τ→0`; this fixed operator + dial is the deliverable.

**Selection / inference (locked firewall):**
- Sweep is unnecessary (`Θ, B` are learned), but selection across candidate `(γ, λ_P, τ-schedule, N_refit)` recipes is done **on val only** via `cmd_canary`, then a single recipe is committed to the locked dev/gate split via `cmd_gate` — identical val→dev governance as every other HQ arm.
- Inference is one forward pass: measure with the frozen binary `A_θ`, `x0 → d_A,d_G → fuse → audit_flat`. No test-time optimization.

## 6. Honest gain assessment — likely MARGINAL over HQ-12, plausibly meaningful over the *default* fixed operators

- **Vs. the current fusion on DCT/Hadamard/random rows: plausibly a real win.** Those operators are prior-agnostic; co-designing the rows for the fusion is a genuinely new lever and Proposition 1 guarantees no regret. Learned-operator precedents (cards #29, #30, #98) all report gains from ending operator-agnosticism.
- **Vs. HQ-12 (perception-weighted operator): likely small.** HQ-12 already moves the perceptually-dominant directions into the row space using the LPIPS Gram. HQ-13's *extra* information is only the fusion-conditional part — "which perceptual directions does the frozen fusion already invent well, so we needn't measure them." Because our fusion is a **shallow two-point blend of two frozen residuals** (not a deep posterior sampler), its "what it can invent" set is limited and largely overlaps with generic natural-image structure, which HQ-12's Gram already captures. The incremental perceptual variance HQ-13 can additionally relocate is bounded and probably second-order.
- **Constraint tax.** The binary/transmission-feasible set `𝒜` and the `m`-row budget restrict how sharply `A_θ` can be steered; the standardization map means `A₀ ∈ 𝒜` is only approximate, so even Proposition 1's dominance can be eroded by a few hundredths of an LPIPS point of optimization slack.
- **Coupling weakness.** With priors frozen, `B` and `A_θ` couple only through the null geometry; the dial has little to co-adapt beyond what a post-hoc `B`-sweep already finds. Most of the lever is in `A_θ`, i.e. HQ-13 ≈ HQ-12 + a fusion-conditional correction.

**Bottom line.** Expected gain over the *current fixed-operator* fusion: plausibly real but modest. Expected gain over HQ-12 specifically: **likely marginal**; HQ-13's honest value is (a) the cleanest *co-design* narrative (operator learned *for* the reconstructor, consistency free-by-construction — a story cards #29/#30/#16 set up but none get for free), and (b) it *stacks on top of* HQ-12 (initialize `Θ` from the HQ-12 perceptual-PCA operator and let co-design add the fusion-conditional correction), rather than competing with it. It should be presented as the co-design capstone of the operator-side thread (HQ-12→HQ-13), not as an independent large-ceiling bet.

## 7. Implementation gaps to close before running (honesty about assets)

1. **Differentiable exact projector in `A_θ`.** `ExactRangeNullProjector` (`src/projections.py`) is currently built from a *fixed* `A` (hashes `A`, caches the Gram factorization). HQ-13 needs `x0_θ`, `P_0^θ` to be differentiable in `A_θ`. Two options: (a) implement the projector with `torch` autograd-tracked `solve_gram` (`(A_θ A_θᵀ + λI)⁻¹` via `torch.linalg.solve`, differentiable) so `A_θ⁺ = A_θᵀ(A_θ A_θᵀ)⁻¹` flows gradients; or (b) reuse `LearnableGhostMeasurementOperator.dc_project / null_project / data_solution` (already differentiable, already gradient-checked) as the differentiable projector during training, then re-audit the frozen operator through the exact `ExactRangeNullProjector` at eval. Option (b) is lower-risk and reuses verified code; it must be checked that the two projectors agree to `≤1e-5` on the final fixed `A_θ` (there is a `calibrate_operator_equivalence.py` harness for exactly this).
2. **LMMSE-under-drift.** `EmpiricalLMMSE` is fit for a fixed `A`. Either re-fit periodically (§5 step 2) or use the noiseless range anchor `A_θ⁺ y` through the differentiable projector to avoid a stale prior. Report which; the noiseless-limit choice keeps everything differentiable and drift-free and is recommended for the clean testbed.
3. **First experiment (matches the deck's "First experiment").** From `src/sanity_learnable_patterns.py` at 5 %: freeze the VQAE/VQGAN priors, attach the differentiable exact-fusion head + LPIPS(+γ MSE) loss, train only `Θ, B` for a few hundred steps on STL10-train; assert `audit_flat` stays at numerical zero throughout; compare **gated** LPIPS/PSNR vs fixed DCT/Hadamard rows *and vs the HQ-12 operator* on seeds 0/1/2. **Success:** `relmeaserr ≤ 1e-5` maintained AND gated LPIPS ≤ the locked `−32.6 %` at equal-or-smaller PSNR cost on ≥ 2/3 seeds, with the honest expectation that the delta **over HQ-12** is small.

## 8. Grounding cards (invented no numbers)

- **#30 Learned Primal-Dual (Adler & Öktem 2018):** end-to-end operator-embedded reconstruction recovers a true feature that operator-agnostic methods render indistinguishable from false ones — the motivating precedent for co-designing sensing with reconstruction.
- **#29 Variational Network (Hammernik et al. 2018):** unrolled forward-operator-in-the-loop training with learned per-step data-term weights; canonical learned-reconstruction family we co-design against.
- **#16 Deep Decomposition Learning (Chen & Davies 2020):** `A = P_r F + P_n G` with `P_r = H⁺H`, `P_0 = I − H⁺H`; establishes that learning inside the R–N split is standard — HQ-13 differs by keeping consistency analytic (no learned/CG data-consistency block) and learning the *operator*, not the null content.
- **#5 NPN (Jácome et al. 2025):** data fidelity leaves the null space uncontrolled; they *learn* null structure. HQ-13 instead *reshapes* the null space by choosing `A_θ`, keeping the governed-dial stance.
- **#98 SPI-DL review (Song et al. 2025):** `B = H T + η`, SPI≡GI congruent, reconstruction quality depends heavily on the modulation matrix `H` at extreme undersampling — the field-level justification for learning `H = A_θ` jointly with the reconstructor.
- **Assets:** `src/sanity_learnable_patterns.py` (verified STE + gradient flow), `src/measurement.py::LearnableGhostMeasurementOperator/LearnablePatternBank`, `src/projections.py::ExactRangeNullProjector` (+ `calibrate_operator_equivalence.py`), `vqgan_detail_fusion.py::{prep_residuals,fuse,radial_masks,apply_band,select_operating_point}`, `hq.EmpiricalLMMSE`, `src/pattern_regularizers.py::total_pattern_loss`, `cmd_canary`/`cmd_gate`.
