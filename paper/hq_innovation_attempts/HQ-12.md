# HQ-12 — Perception-Weighted Operator Design: Putting the Row Space Where LPIPS Lives

**Status:** proposed (paper-ready method subsection + one proposition + ablation design; no numbers invented).

**One-line.** Choose the `m` measured ghost-imaging rows to span the directions of maximum *perceptual* (LPIPS) variance, so the fixed 2–10 % measurement budget already captures the semantically dominant content and the null space we must invent becomes *perceptually cheap*.

---

## 1. Position within the program and the invariant it respects

Every other point in this deck (HQ-1…HQ-11, HQ-13/14) accepts the operator `A` as given and asks *how to fill `Null(A)` better*. HQ-12 is the one lever that moves the boundary itself: it **redesigns `A`** so that less of what humans perceive is left to the null space in the first place. It is therefore orthogonal to, and stacks with, every null-space filler — any downstream method (scalar `B` fusion, PnP-RED, MoDL, diffusion) starts from a higher perceptual floor.

**Exact-consistency invariant is untouched.** HQ-12 changes only *which* rows `A` has, not the reconstruction algebra. The row space is still measured exactly and the null space is still edited inside `P0`:

```
x_hat_B = x0 + P0( d_A + B (d_G − d_A) ),   A P0 = 0  ⇒  A x_hat_B = y   (to ~2e-15).
```

The new `A` plugs straight into `get_exact_projector` (`src/projections.py`); `P_R`, `P0`, `data_anchor_flat`, `audit_flat` and the fusion are byte-for-byte unchanged. Because a perceptual-PCA `A` has orthonormal rows by construction (Sec. 4), the projector even takes its `row_orthonormal` fast path (`solver="row_orthonormal"`, `solve_gram = identity`), so consistency is *cheaper* to enforce, not harder.

## 3. Method

### 3.1 Setup and notation

Images `x ∈ ℝ^n` (`n = 64·64 = 4096`, STL10 grayscale/flattened as in `GhostMeasurementOperator`). A structured operator `A ∈ ℝ^{m×n}` with `m = ⌈r·n⌉` at rate `r ∈ {2 %, 5 %, 10 %}`. Let `φ_LPIPS(x) ∈ ℝ^F` be the concatenated, unit-normalized deep-feature embedding used by LPIPS (the same VGG/AlexNet features `metrics.py` already calls); LPIPS between `x, x'` is (a learned-weighted) `‖φ(x) − φ(x')‖`.

### 3.2 The perceptual Gram

The distortion HQ cares about, for a small perturbation `δ` around an operating image `x`, is
```
LPIPS(x, x+δ)²  ≈  δᵀ J(x)ᵀ W J(x) δ ,     J(x) = ∂φ_LPIPS/∂x |_x ,
```
with `W` the (diagonal, non-negative) LPIPS channel weights. Averaging the local metric tensor over the data distribution defines the **perceptual Gram**
```
G  =  E_{x∼D} [ J(x)ᵀ W J(x) ]   ∈  ℝ^{n×n},   symmetric PSD.
```
`G` is the image-space quadratic form whose top eigenvectors are the directions in which perturbations are, on average, *most perceptually visible*.

### 3.3 Perceptual-PCA operator (primary variant, no training)

1. **Estimate `G`.** For a few hundred STL10 train images, form `J(x_i)` by reverse-mode autodiff of `φ_LPIPS` (or a Jacobian-free estimator, Sec. 3.6), accumulate `G_hat = (1/N) Σ_i J_iᵀ W J_i`. `G_hat` is `n×n = 4096²` — trivially storable/eigendecomposable at 64×64.
2. **Top-`m` generalized eigenvectors.** Solve `G v = λ v` and take the `m` eigenvectors with largest `λ`: `V_m = [v_1,…,v_m] ∈ ℝ^{n×m}` (orthonormal, since `G` is symmetric).
3. **Rows of `A`.** Set `A = V_mᵀ`. These are the top-`m` **perceptual principal components** of image space. Rows are orthonormal ⇒ `A Aᵀ = I` ⇒ projector uses its `row_orthonormal` path and `data_anchor_flat(y) = Aᵀ y`.
4. **(Optional) DMD realizability.** GI patterns are physically non-negative and often binary. Binarize each row with a straight-through estimator (`sign(v_j)` forward, identity backward) as `sanity_learnable_patterns.py` already does; re-orthonormalize (QR) so the exact-projector fast path and `A P0 = 0` are preserved. Report both the ideal (real-valued) and DMD-realizable (binarized) operators.

Everything downstream — LMMSE anchor `x0` (`hq.EmpiricalLMMSE`), `d_A`/`d_G`, `fuse()` — is recomputed against the new `A` with **no retraining of any prior**, because the priors condition on `(x0, y, A)` which are all recomputed for the swapped operator by the existing measurement pipeline (`eval.make_measurement`, `reconstruct_from_measurements`).

### 3.4 Whitened / measurement-noise-aware variant

If a noise budget `σ²` is on `y`, weight the eigen-selection by predictability rather than raw perceptual variance, mirroring GSNR's `ρ_j²` (card #2). Trade raw perceptual variance `λ_j` against how reliably that direction is measured under noise: rank rows by `λ_j · c_j/(c_j+σ²)` where `c_j` is the LMMSE prior variance of mode `j`. This keeps `A` from spending its budget on high-perceptual-variance but noise-swamped directions. On the current noiseless testbed this reduces to §3.3.

### 3.5 Perceptual-vs-pixel contrast (the ablation's whole point)

The natural baselines are (i) fixed DCT rows, (ii) Hadamard rows, (iii) random rows, and — the decisive comparator — (iv) **pixel-PCA rows** (card #91's `X = UΣVᵀ` masks). HQ-12 is *only* interesting if perceptual-PCA beats pixel-PCA: both are top-`m` SVD bases, differing solely in whether variance is measured in pixel space (`G_pix = E[x xᵀ]` after centering) or in LPIPS-feature space (`G`). The gap between (iii)/(iv) and perceptual-PCA is the *entire claimed effect*.

### 3.6 Jacobian-free fallback (cost control)

`J(x)` is `F×n`; for the whole VGG stack `F` is large. Two ways to avoid materializing it:
- **Hutchinson/JVP estimator.** `G ≈ E_{x,δ}[ (‖φ(x+εδ)−φ(x)‖_W / ε)² · δδᵀ ]` with Rademacher `δ` and small `ε` — needs only forward passes of `φ` and never forms `J`.
- **Random-projection sketch.** Estimate the top-`m` eigenspace of `G` by a randomized range finder using only JVPs `J v` (forward-mode) / VJPs `Jᵀ u` (reverse-mode), never the full `G`.

Both make the estimate a single eval-time pass over a few hundred images. No prior or refiner is trained.

## 4. Proposition (perceptual advantage is real but linearized, and exact consistency is free)

**Proposition (Perceptual row-space optimality under local linearization).**
Fix rate `m`. Let `G = E_{x}[J(x)ᵀ W J(x)]` be the perceptual Gram (§3.2) with eigenpairs `(λ_1 ≥ … ≥ λ_n, v_1,…,v_n)`. For an operator with orthonormal rows `A = Uᵀ` (`U ∈ ℝ^{n×m}`, `UᵀU = I_m`), the null projector is `P0 = I − U Uᵀ`, and the **expected linearized perceptual energy left in the null space** is
```
E_perc(U)  :=  E_x[ ‖ W^{1/2} J(x) P0 δ_x ‖² ]  for the worst-case unit null perturbation
            =  λ_{max}( P0 G P0 ).
```
The minimizer over all rank-`m` orthonormal `U` is `U⋆ = V_m = [v_1,…,v_m]` (the top-`m` perceptual eigenvectors of §3.3), and then
```
min_U  λ_max(P0 G P0)  =  λ_{m+1}(G) .
```
Any fixed basis whose span is not aligned with `V_m` (DCT, Hadamard, random, or pixel-PCA when `G_pix ≠ G`) leaves a *strictly larger* worst-case perceptual null energy `λ_max(P0 G P0) > λ_{m+1}(G)`. Simultaneously, because `A Aᵀ = I`, `A x_hat_B = y` holds exactly for every `B` (unchanged from the current fusion).

*Proof sketch.* `P0 G P0` restricted to `Null(A)` is a rank-`(n−m)` PSD form; by Courant–Fischer its largest eigenvalue is minimized over `m`-dim complements exactly when the removed subspace is the top-`m` eigenspace of `G`, giving the `(m+1)`-th eigenvalue. Orthonormal rows give `A Aᵀ = I` so `data_anchor_flat` and `P0 = I − AᵀA` are exact; the fusion consistency proof is unchanged. ∎

**This is a linearization result, and the manuscript must say so.** `J(x)` is image-dependent (LPIPS is nonlinear), so a *single fixed linear* `A` cannot make the *nonlinear* perceptual null cost vanish; the proposition bounds only the **average local** perceptual energy under the Gram `G = E_x[JᵀWJ]`. The honest claim is therefore: perceptual-PCA rows are optimal for the *average linearized* perceptual metric, which is a principled surrogate for LPIPS, not LPIPS itself. This gap is precisely why the effect could be marginal (Sec. 6) and must be measured, not assumed.

## 5. Experiment / ablation design

**Testbed.** STL10 64×64, structured operators at 2/5/10 %, locked val→dev multi-seed gate (`cmd_canary`/`cmd_gate`), seeds 0/1/2. Metrics: LPIPS (primary), PSNR (budget), RAPSD (texture realism), plus `relmeaserr` for the consistency audit.

**Harness.** The operator-swap path already exists: `src/eval_pattern_swap.py` and `src/calibrate_operator_equivalence.py` swap `A` while holding everything else fixed; `src/sanity_learnable_patterns.py` provides the straight-through binarization for the DMD-realizable variant. HQ-12 adds only a `build_perceptual_pca_operator(rate, n_images)` that returns an `A` for `GhostMeasurementOperator(..., matrix_normalization="orthonormal_rows")`.

**Arms (each at 2/5/10 %, identical fusion and priors downstream):**
1. DCT rows (current locked baseline).
2. Hadamard rows.
3. Random rows.
4. **Pixel-PCA rows** (card #91 masks) — the decisive comparator.
5. **Perceptual-PCA rows** (HQ-12, §3.3), real-valued.
6. **Perceptual-PCA rows, DMD-binarized** (§3.3 step 4).
7. Noise-aware perceptual rows (§3.4) — only on the noisy-`y` operator.

**Primary readout.** For each arm run the *unchanged* fusion (`x0 → d_A,d_G → fuse → audit_flat`) and select `B` on the locked gate. Report the gate LPIPS-vs-PSNR Pareto per arm and per rate.

**Pre-registered success criteria.**
- **Consistency (must pass for all arms):** `relmeaserr ≤ 1e-5` after `audit_flat`; verify `A Aᵀ = I` to tolerance so the `row_orthonormal` path is taken.
- **Perceptual-null shrinkage (mechanism check, prior-free):** empirically `λ_max(P0 G P0)` for arm 5 ≤ that of arms 1–4 on held-out train images — i.e. the proposition's quantity actually decreases. This is measurable *before any reconstruction* and is the cleanest evidence the mechanism fires.
- **End-to-end win:** arm 5 beats arms 1–4 (especially arm 4, pixel-PCA) on gate LPIPS at equal-or-smaller PSNR cost, on ≥2/3 seeds, at ≥1 rate (expect the largest gap at the lowest rate 2 %, where the null space is largest).
- **DMD realizability:** arm 6 retains most of arm 5's gain after binarization+QR.

**Figures.** (a) perceptual eigen-spectrum of `G` with the `m`-cutoff marked at each rate; (b) example rows of perceptual-PCA vs DCT/pixel-PCA; (c) `λ_max(P0 G P0)` bar chart across arms (the mechanism figure); (d) gate LPIPS-vs-PSNR Pareto overlay, perceptual-PCA vs baselines.

**Cost.** One Jacobian/JVP pass over a few hundred images to build `G` (§3.6) + one eval pass per arm through the existing gate. No prior or refiner training. Comparable cost to HQ-7/8/9 near-term wins.

## 6. Honest gain assessment: large or marginal?

**Where it could be large.** At the lowest rate (2 %) the null space is `~98 %` of dimensions, so *how* the `2 %` is spent has maximal leverage; card #91 already shows a data-driven SVD basis (pixel-PCA) sharply beating random/Hadamard/Hartley masks in exactly this GI regime, and card #2-deck (HalluGen) confirms perceptual error concentrates in the null space. If perceptual variance is *anisotropic and low-dimensional* (a handful of eigenvectors dominate `G`), spending the budget on those eigenvectors could lift the LPIPS floor for **every** downstream filler at once — a rare "raise-the-floor-for-all" gain rather than a per-method delta.

**Where it is likely marginal — stated plainly.** Three concrete reasons the gain over the *current fusion* may be small:
1. **DCT already correlates with perceptual variance.** Natural-image energy and perceptual salience both concentrate in low spatial frequencies; the current DCT operator is *already* a near-perceptual basis. The realistic delta is perceptual-PCA vs DCT — plausibly a few-percent LPIPS move, not a step change. The clean, prior-free evidence is the arm-4-vs-arm-5 gap (pixel-PCA vs perceptual-PCA); if *that* is small, the perceptual weighting is not buying much beyond ordinary PCA.
2. **Linearization gap (Sec. 4).** A single fixed linear `A` optimizes only the *average local* perceptual metric. The true LPIPS null cost is image-dependent; a per-image optimal row space is unreachable with a fixed operator, capping the achievable gain.
3. **The information caveat (card #91, load-bearing).** Reorienting `A` does **not** add information — it only reallocates which content is measured vs invented. It cannot manufacture perceptual quality; it can only stop the budget from being spent on perceptually-cheap directions. If the baseline operator is not badly misaligned, there is little to reclaim.

**Net honest verdict.** HQ-12 is a **near-term win with a modest expected LPIPS delta over the current DCT fusion, but unusually high strategic value**: it is the *only* lever that shifts the perceptual floor for every other point in the deck simultaneously, it is prior-free and cheap, and it comes with a clean, pre-registration-friendly mechanism check (`λ_max(P0 G P0)`) that can *falsify the whole idea before any reconstruction is run*. Recommended framing in the paper: report it as an **operator-design ablation that stacks with the null-space fillers**, foreground the mechanism figure and the pixel-PCA-vs-perceptual-PCA contrast, and do **not** claim a large end-to-end jump unless arm-5-vs-arm-4 shows one. The bigger operator-side bet (learning `A` jointly *for our fusion*, closing the linearization gap by co-design) is HQ-13; HQ-12 is its training-free, analyzable precursor.
