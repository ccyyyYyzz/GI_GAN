# HQ-03 — Codebook-lookup null-space refiner (CodeFormer-style global code prediction)

*Realization of HQ Innovation Point 3 as a paper-ready **Method** subsection: precise formulation + architecture + train/inference recipe + ablation design + one proposition. Grounded in `GI99_reading_cards.json` (CodeFormer, VQGAN/Taming, ViT-VQGAN, VQ-VAE, VQ-VAE-2, VQFR, RestoreFormer, Perception–Distortion) and in the actual repo code path that produces `d_G` today.*

**Strengthens.** HQ ceiling (raise the LPIPS floor at 2–5% sampling) — the detail-branch `d_G` generator only.
**Invariant held throughout.** The fusion path is unchanged: `x̂_B = x0 + P0(d_A + B(d_G − d_A))`, `A P0 = 0`, so `A x̂_B = y` to ~2e-15 for every `B`. HQ-03 changes only *how `d_G` is generated*; it buys quality strictly inside `Null(A)` and relaxes no consistency guarantee.

---

## 1. The problem, stated against the actual code path

Today's detail residual is produced by `regen_seed().refine(VQGAN, x0, unc)` in `vqgan_detail_fusion.py`, which is:

1. `z0 = prior.model.encode(x0)` — encode the **LMMSE anchor `x0`** (a heavily corrupted 2–5% GI reconstruction) to the token-grid feature map `z0 ∈ R^{B×C×h×w}` (here `h×w` is the VQAE/VQGAN latent grid).
2. `dz, dlogits = refiner(x0, unc, z0)` — a per-token perturbation.
3. `logits = logits_from_latent(z0 + dz, prior, …) + dlogits`, where `logits_from_latent` (in `anchor_initialized_vqgan_inversion.py`) computes, **independently at each token**, the negative squared distance to every codebook vector: `−‖z_t − e_k‖²` → a softmax over `k`. This is exactly VQ-VAE nearest-neighbour lookup `z = argmin_k ‖z_e(x) − e_k‖` [VQ-VAE card] softened by temperature.
4. `zq = quantize_from_logits(...)`; `x_G = decode_embeddings(zq)`; then `d_G = P0(x_G − x0)` in `prep_residuals`.

The failure mode is structural and local: **each token's code is chosen from that token's own corrupted feature**, with no mechanism to let uncorrupted neighbours or global image composition override a wrong local match. At 2–5% GI the anchor `x0` is precisely where local features are least reliable, so `argmin_k` returns wrong codes and `x_G` — hence `d_G` — carries token-local artefacts exactly where we most need clean detail. The refiner's `dz, dlogits` are themselves per-token corrections and inherit the same locality.

This is the *documented* weakness that CodeFormer was built to fix. CodeFormer's core finding [CodeFormer card, R059]: "because nearest-neighbour code lookup fails when LQ textures are corrupted, a Transformer predicts the code sequence from LQ features (global composition), letting the network reduce reliance on the degraded input"; the discrete finite prior (~1024 codes) "attenuates LQ-HQ mapping uncertainty and gives superior robustness to heavy degradation vs. continuous generative priors (PULSE, GFP-GAN)." The mechanism we currently use is the one CodeFormer reports as brittle; the mechanism CodeFormer reports as robust is not yet in our pipeline.

**Claim of this point (honestly scoped).** Replacing per-token distance lookup with a small global code-prediction transformer should produce a cleaner, more semantically coherent `x_G`, lifting LPIPS **most at the lowest rate (2%) where lookup breaks**, and **little at 10%** where local features are already good enough that `argmin_k` is mostly right. See §6 for why the *expected* gain over the tuned fusion is real but modest-to-moderate, not large.

## 2. Precise formulation

Let the frozen VQGAN comprise encoder `E`, codebook `Z = {e_k}_{k=1}^{K} ⊂ R^{d}`, and decoder `G` (`decode_embeddings`). For an anchor `x0`, write the token grid `z0 = E(x0) ∈ R^{h×w×d}` and let `s ∈ {1,…,K}^{h×w}` be a code-index sequence over the `N = h·w` tokens.

**Current (local) index law.** `s_t^{loc} = argmax_k [ −‖z0_t − e_k‖² / τ ]` (softened; `logits_from_latent`). Each `s_t` depends only on `z0_t`.

**Proposed (global) index law.** A transformer `T_φ` predicts a distribution over the whole sequence jointly:

> **p_φ(s | z0) = ∏_{t=1}^{N} softmax_k( ℓ_φ(z0)_{t,k} ),   ℓ_φ(z0) = T_φ( PatchEmbed(z0) + PosEmb ) ∈ R^{N×K}.**

The token logits `ℓ_φ(z0)_{t,·}` are a function of the **entire** `z0` (self-attention across all tokens), not of `z0_t` alone. At inference take `ŝ_t = argmax_k p_φ(s_t | z0)` (or low-temperature sampling), form `ẑ_q = [ e_{ŝ_t} ]_t` via the existing `quantizer.lookup_indices`, decode `x_G = G(ẑ_q) = decode_embeddings(ẑ_q)`, and set, unchanged,

> **d_G = P0( x_G − x0 )   (via `null_project_flat` in `prep_residuals`).**

Everything downstream (`fuse`, `apply_band`, `select_operating_point`, `audit_flat`, the gate) is byte-for-byte identical; only the producer of `x_G` is swapped.

**Second, orthogonal knob (Controllable Feature Transformation).** CodeFormer exposes a scalar `w ∈ [0,1]` via a Controllable Feature Transformation (CFT) that affinely blends fidelity-bearing encoder features `F_e` into the decoder features `F_d`: `F̂_d = F_d + (α·F_d + β)·w`, `w=0` max-quality/robustness, `w=1` max-fidelity [CodeFormer card]. We port CFT to gate **how much of the anchor's local feature `z0` is re-injected before decoding**:

> **ẑ_q(w) = ẑ_q + w · ( E_skip(z0) − ẑ_q ),**   `w ∈ [0,1]`,

where `E_skip` is a 1×1 conv adapter. `w` is **orthogonal to `B`**: `w` shapes the *content* of `x_G` (how anchor-anchored vs. codebook-clean the detail is) *before* projection; `B` (and the radial band schedule) governs *how much* of the resulting null residual is admitted. Both are pre-registered scalars swept on val and locked on the gate. Crucially `w` operates entirely upstream of `P0`, so — unlike CodeFormer's `w`, which moves the whole reconstruction off the measurement — **`A x̂ = y` is invariant to `w` here**: `w` never buys or costs measurement fidelity, it only re-shapes null content (this contrast is the paper-grade differentiation, §5).

## 3. Architecture (sized for the 64×64 STL10 testbed)

- **Backbone.** A GPT/ViT-style encoder-only transformer `T_φ` over the `N = h·w` latent tokens (`h=w=8` for the current 64×64 grid at the VQAE/VQGAN downsampling → `N=64` tokens; confirm from the loaded checkpoint's `z0` shape). PatchEmbed = 1×1 conv `d → D` (`D≈256`), learned positional embedding, **2–4 self-attention blocks**, 4 heads, MLP ratio 2. Output head: linear `D → K` giving per-token logits over the frozen codebook of size `K` (the VQGAN codebook, unchanged). Total ≈ 0.5–2 M params — the "small GPT-style transformer" the deck specifies, dwarfed by the frozen decoder.
- **Frozen throughout.** `E`, `Z`, `G` (the whole VQGAN) and the VQAE structure branch. Only `T_φ` (and the CFT adapters `E_skip`, `α`, `β`) train. The existing per-token refiner (`dz, dlogits`) is *removed* on the VQGAN branch and replaced by `T_φ`; the VQAE structure branch (`d_A`) is untouched.
- **Codebook hygiene (borrowed, optional).** If codebook usage is low, adopt ViT-VQGAN's two fixes at *codebook* level without retraining `G`: factorized low-dim lookup and **L2-normalized codes** (cosine-similarity lookup), which "raise codebook usage even at size 8192" and kill dead codes [ViT-VQGAN card]. Since HQ-03 predicts indices rather than looking them up, this mainly stabilizes the training target distribution; apply only if a usage histogram shows collapse.
- **Where it lives.** `refiner_ckpt(seed, VQGAN)` / `load_refiner_checkpoint` / the `refine(VQGAN, …)` closure already host a swappable per-branch module. `T_φ` is dropped in behind that interface; `refiner_ckpt` gains a `code_predictor` variant. No change to `regen_seed`'s caching contract (`x_G` still lands in `pack["x_G"]`).

## 4. Training recipe (self-supervised on HQ code grids; no GT-null supervision)

The supervision is CodeFormer's stage-II code-prediction objective, adapted:

1. **Targets (offline, once).** Take ~2000 STL10 64×64 **HQ** (clean) images `x*`; encode each with the frozen `E` and take the *teacher* code grid `s* = argmin_k ‖E(x*)_t − e_k‖` — the codes the frozen VQGAN itself assigns to the clean image. These are the "correct" indices lookup on the corrupted anchor gets wrong.
2. **Inputs.** The matched anchor `z0 = E(x0)`, where `x0 = LMMSE.anchor(A x*)` is the operator's own anchor for that image (so the train distribution matches `regen_seed`).
3. **Losses.**
   - **Code cross-entropy** `L_code = Σ_t CE( p_φ(s_t | z0), s*_t )` — the primary target (predict the clean code sequence from the corrupted grid). This is CodeFormer's discrete-prediction loss; it is what makes global prediction robust to corruption.
   - **Image/perceptual** `L_img = ‖G(ẑ_q) − x*‖₁ + λ_lp·LPIPS(G(ẑ_q), x*)` reusing `image_loss`/`differentiable_lpips` already in the repo, with straight-through through `quantize_from_logits(..., straight_through=True)` so gradients reach `T_φ`. This ties code prediction to decoded quality (VQGAN's perceptual-codebook rationale [VQGAN card]).
   - **CFT `w` schedule.** Sample `w ∼ U[0,1]` per batch so a single `T_φ` supports the whole `w` sweep at inference (CodeFormer trains CFT across `w`).
   - Total `L = L_code + λ_img L_img`, `λ_img ≈ 0.25` (mirrors `lambda_lpips`). No adversarial term (the frozen decoder already carries VQGAN's GAN-trained realism).
4. **Budget.** A few hundred–2000 steps, one branch, ≈ tens of minutes on one GPU — comparable to the existing refiner training. Train **per seed** (0/1/2) exactly like the current refiners so multi-seed gating is apples-to-apples.
5. **Firewall.** Train on the **train** split only; select `w` (and any decode temperature) on **val**; report on the locked **dev/gate** — the same val→dev firewall (`cmd_canary`/`cmd_gate`) the fusion arms already use. `T_φ` never sees dev.

## 5. Inference recipe

Per image, per seed: `z0 = E(x0)` → `ŝ = argmax_k T_φ(z0)` → CFT with the selected `w*` → `ẑ_q` → `x_G = decode_embeddings(ẑ_q)` → `d_G = P0(x_G − x0)` → `fuse` with the **already-locked** `B*`/band schedule → `audit_flat`. One forward pass through a ~1 M-param transformer + the frozen decoder; inference cost ≈ the current refiner. Consistency check `relmeaserr ≤ 1e-5` must hold by construction; keep the final `audit_flat` guard against float drift.

**Differentiation to bank (Related-Work / Method contrast).** CodeFormer's `w` and our reuse of it look identical at the interface but differ categorically: CodeFormer's CFT re-injects fidelity features into the *final decoder*, so higher `w` moves the whole image and genuinely trades measurement fidelity for quality [CodeFormer card, innovation_spark]. Our `w` acts on `x_G` *before* `P0`; the projector then discards whatever `w` added inside `Range(A)`. So `A x̂ = y` for **every** `(B, w)` — a "dial that costs nothing on the measurement," which is the same categorical (not gradual) distinction the paper already draws for `B` vs. CodeFormer's `w`. RestoreFormer/VQFR are cited as the cross-attention / parallel-decoder lineage of "fuse degraded-input fidelity with an HQ VQ dictionary" [RestoreFormer, VQFR cards]; HQ-03 sits in that lineage but is the only member whose fidelity/detail boundary is an exact operator split rather than an architectural heuristic (VQFR's parallel decoder and compression-patch knob exist precisely to stop this "contamination" heuristically [VQFR card]).

## 6. Ablation / experiment design (and the honest gain call)

**Primary head-to-head (the deck's first experiment).** Detail-branch variant `code_predictor` vs. `lookup`, everything else fixed:

- **A0 (baseline)** current `refine(VQGAN)` = local-distance logits + per-token refiner → `d_G^{loc}`.
- **A1 (HQ-03)** `T_φ` global code prediction, `w` fixed at the CFT default (0.5) → `d_G^{glob}`.
- **A2 (HQ-03 + CFT)** A1 with `w*` selected on val.
- Run `cmd_canary` at **2% seed0** first (where the effect should be largest), then 5% and 10%; compare **LPIPS and PSNR at matched `B`** (reuse the locked `B*`), then re-select `B` per arm and compare the whole P–D Pareto via `select_operating_point`. Promote to `cmd_gate` on seeds 0/1/2 only if 2% wins.
- **Success criterion (pre-registered).** At 2%: LPIPS strictly below the lookup arm at PSNR drop ≤ 0.5 dB, on ≥2/3 seeds through the locked gate, with `relmeaserr ≤ 1e-5`. Neutral-or-win at 10% (no regression where lookup is already adequate).

**Diagnostic ablations (explain, don't just score).**
1. **Code-agreement rate** `mean_t 1[ŝ_t = s*_t]` for lookup vs. `T_φ`, as a function of rate (2/5/10%). The mechanistic prediction — global prediction recovers more clean codes at 2% than at 10% — is directly testable and *is the causal story* for any LPIPS delta. If agreement rises but LPIPS doesn't, the fusion/`B` is absorbing the gain (report honestly).
2. **`w`-sweep flat-residual figure.** Sweep `w ∈ {0,…,1}` and plot `‖A x̂ − y‖` (flat ~2e-15) alongside LPIPS — the visual proof that our `w` is a zero-measurement-cost knob, the head-to-head against CodeFormer's drifting `w`.
3. **Locality control.** Restrict `T_φ` to 0 attention blocks (per-token MLP only) → should ≈ recover lookup, isolating the *global-composition* contribution from mere extra capacity (guards against the deck's own risk that a learned gate re-learns the local baseline).
4. **Codebook-usage histogram** with/without ViT-VQGAN L2-norm codes, to confirm prediction isn't collapsing onto a few codes.

**Honest gain assessment (required by the brief).** **Expected gain over the current tuned fusion: moderate at 2%, small at 5%, ~zero at 10% — not large.** Reasons, stated plainly:

- The gain is bounded by how many codes lookup actually gets wrong. At 10% the anchor features are good enough that `argmin_k` is mostly right, so `x_G` is already near-clean and there is little for `T_φ` to fix. The upside concentrates at 2%.
- Whatever `T_φ` improves in `x_G` is then **projected through `P0` and mixed by `B`** into an already-well-tuned residual. The locked `−32.6% LPIPS` operating point was selected *with* the noisy lookup `d_G`; a cleaner `d_G` only helps to the extent the current `B*` was leaving quality on the table because the residual was untrustworthy. If the tuned `B` had already learned to down-weight noisy high-frequency `d_G`, a cleaner `d_G` mostly lets `B` rise — a second-order win, not a new capability.
- Unlike HQ-01 (diffusion posterior), HQ-03 does **not** enlarge the reachable set with genuinely new detail content; it makes the *same* codebook-decoded detail cleaner. It attacks the deck's stated *root cause* (wrong codes at low rate) but within the existing two-residual fusion envelope.
- Net: HQ-03 is the **highest-value single-arm upgrade that keeps the frozen decoder and the exact-consistency invariant**, and its 2% result is the crisp, defensible claim ("global code prediction recovers the LPIPS lookup leaves on the table at the lowest sampling rate"). Overselling it as a large uniform gain would not survive the 10% row. The mechanistic agreement-rate figure (diagnostic 1) is what makes even a modest LPIPS delta a *publishable, explained* result rather than a lucky tuning artefact.

## 7. Proposition (why global prediction can only help the null residual, and why it's bounded)

**Proposition (global code prediction is a strict superset that stays exactly consistent).**
Let `E, Z, G` be the frozen VQGAN and `P0 = I − A†A` the exact null projector (`A P0 = 0`). Let `s^{loc}(z0)` be the per-token argmin index law and `p_φ(·|z0)` the transformer index law of §2, with decoded images `x_G^{loc} = G(z_q(s^{loc}))` and `x_G^{glob} = G(z_q(ŝ))`, `ŝ = argmax p_φ`. Then:

1. **(Consistency invariance.)** For any index law `s` and any `B, w`, `A x̂_B = y` exactly, because `d_G = P0(x_G − x0)` and `A P0 = 0` regardless of how `x_G` was produced. Swapping lookup for `T_φ`, and sweeping `w`, cannot change the measurement residual. [VQ-VAE lookup; VQGAN decoder; exact `P0`.]
2. **(Representational superset.)** The family `{p_φ}` contains the local law: setting `T_φ` to zero attention blocks and its head to `ℓ_{t,k} = −‖z0_t − e_k‖²/τ` reproduces `s^{loc}` token-for-token. Hence, at its optimum on `L_code`, `T_φ` attains code-agreement with the clean teacher `s*` **no worse than lookup**: `E[ 1(ŝ_t=s*_t) ] ≥ E[ 1(s^{loc}_t=s*_t) ]`. Global prediction can only match-or-raise the fraction of correct clean codes.
3. **(Bounded payoff.)** The LPIPS gain is bounded by the decoder's sensitivity to the corrected tokens: `LPIPS(x_G^{glob}, x*) − LPIPS(x_G^{loc}, x*) ≥ −L·‖z_q(ŝ) − z_q(s^{loc})‖`, and the corrected-token set shrinks as the sampling rate rises (fewer wrong local codes to fix). Therefore the *expected* benefit is monotone-decreasing in rate — largest at 2%, vanishing as lookup becomes reliable.

*Proof sketch.* (1) is `A P0 = 0` applied to any `x_G`, and `w` acts only pre-`P0`. (2) is the deck's own construction: the transformer's hypothesis class strictly contains the distance-logit map (a fixed 1×1 head with no attention), so its empirical-risk minimizer over `L_code` dominates the fixed local map on the training objective; the agreement inequality follows because `L_code` *is* the (soft) code-agreement objective. (3) is Lipschitz continuity of `G` and `LPIPS` composed with the fact that `ŝ` and `s^{loc}` differ only on tokens where local lookup was wrong, whose count decreases with rate (§6 diagnostic 1). ∎

**Reading.** Global code prediction is a consistency-preserving, provably-no-worse replacement for local lookup whose *realized* value is concentrated exactly where the deck predicts — the low-rate regime — and is honestly bounded above by the decoder's response to the finite set of codes lookup gets wrong. The proposition licenses the claim "can only help, held exactly consistent" without licensing "helps a lot everywhere."
