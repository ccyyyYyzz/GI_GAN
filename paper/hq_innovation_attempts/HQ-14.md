# HQ-14 ‚Äî Learned Primal‚ÄìDual two-stream refiner that also cleans the range space (noisy-`y` regime)

**Status: proposed** (paper-ready method subsection + proposition + experiment/ablation design; not yet run).

**Scope note (honest, up front).** This is deliberately *not* a clean-`y` quality lift. On the current noiseless
64√ó64 STL10 testbed the exact null-only fusion (`A x_hat = y` to ~2e-15) is provably as good as or better
than anything that relaxes consistency, so HQ-14 **cannot and should not** beat the locked `‚àí32.6% LPIPS`
operating point there. HQ-14 earns its place only once a *noisy / real-GI* operator is on the table, where the
exact-consistency invariant becomes a liability: it locks measurement noise `A‚ÄÝe` into the reconstruction and
caps PSNR. The expected gain over the current fusion is therefore **regime-conditional and potentially large in
the noisy regime, and zero-to-negative in the noiseless regime** ‚Äî see ¬ß6 for the honest accounting. This point
runs as a **separate soft-consistency arm** and does not touch the locked exact-fusion line.

---

## 1. Problem statement: why exact consistency is wrong under noise

Our whole program buys quality inside the exact null projector. With operator `A` (structured DCT/Hadamard/random
rows at 2/5/10%), min-norm range anchor `A‚ÄÝy`, and projectors `P_R = A·µÄ(AA·µÄ)‚Åª¬πA`, `P_0 = I ‚àí P_R` (matrix-free in
`src/projections.py`: `row_project_flat`, `null_project_flat`, `data_anchor_flat`, `audit_flat`; `A P_0 = 0`), the
fusion writes

```
x_hat_B = x0 + P_0( d_A + B (d_G ‚àí d_A) ),      A x_hat_B = y  exactly for every B.
```

Under **clean** measurements `y = A x*` this is optimal geometry: the measured coordinates are ground truth, and
all freedom lives in `Null(A)`, so quality can only be bought there.

Under **noisy** measurements `y = A x* + e`, `e ~ N(0, œÉ¬≤ I_m)` (GI read/shot noise on the bucket detector), the
*same* geometry becomes a defect. Any estimator pinned to `A x_hat = y` satisfies

```
P_R x_hat = A‚ÄÝ y = A‚ÄÝ A x* + A‚ÄÝ e = P_R x* + A‚ÄÝ e,
```

i.e. it reproduces the true range component **plus the back-projected measurement noise `A‚ÄÝ e`**, which it can
never remove because removing it would violate `A x_hat = y`. The range-space error `‚ÄñA‚ÄÝ e‚Äñ` is a hard PSNR floor
that *no* null-space filler ‚Äî however good `d_A`, `d_G`, or a diffusion posterior ‚Äî can cross. Crucially, our
current LMMSE anchor inherits this exactly: `EmpiricalLMMSE.anchor()` (in `gan_high_quality_gi.py`) ends with
`projector.audit_flat(pred, y)`, which **re-pins the LMMSE prediction's measured modes back onto the noisy `y`**,
discarding the denoising the LMMSE regression would otherwise do in the range. So today, even the anchor carries
`A‚ÄÝ e`.

The Perception‚ÄìDistortion tradeoff (R061 / card #50) already tells us the *distortion*-optimal reconstruction
under noise is the posterior mean `E[x* | y]`, whose range component is the **shrunken** (denoised) measurement,
not the raw `A‚ÄÝ y`. Our exact fusion forgoes exactly that shrinkage. HQ-14 restores it.

## 2. Method: two-stream range+null decomposition, unrolled as a Learned Primal‚ÄìDual net

We generalize the reconstruction from *null-only* to a **physics-gated two-stream residual**, following the Deep
Decomposition Learning template (R022 / card #15), `A(y) = A‚ÄÝy + P_R F(¬∑) + P_0 G(¬∑)`, but instantiated on our
matrix-free projector and unrolled as a Learned Primal‚ÄìDual (LPD) network (R040 / card #29):

```
x_hat = A‚ÄÝy  +  P_R ¬∑ F_œÜ(x0, y)  +  P_0 ¬∑ G_œà(d_A, d_G, x0).      (Eq. HQ14-1)
        ‚îî‚îÄ‚îÄ range denoiser ‚îÄ‚îÄ‚îò      ‚îî‚îÄ‚îÄ null detail refiner ‚îÄ‚îÄ‚îò
```

- **Null stream `G_œà`** = our existing frozen detail machinery. In its simplest form `G_œà(d_A,d_G,x0) = d_A +
  B(d_G ‚àí d_A)` recovers today's fusion verbatim; more generally it is the frozen multi-seed VQAE (`d_A`) + VQGAN
  (`d_G`) refiners (`refiner_ckpt`, `prep_residuals`, `apply_band`, `weight_map`). `G_œà` is what invents detail.
- **Range stream `F_œÜ`** = a *new, small, learned* residual net that estimates and subtracts the measurement-noise
  component living in the range. It is the only genuinely new trainable block. It removes `A‚ÄÝ e` that null-only
  fusion is forced to keep.

Because `P_R` and `P_0` are orthogonal complements, the two streams do not interfere geometrically: `F_œÜ` moves
the estimate only inside `Range(A·µÄ)` (the measured coordinates), `G_œà` only inside `Null(A)`. The exact-fusion
line is the special case `F_œÜ ‚â° 0`.

### 2.1 Unrolled Learned Primal‚ÄìDual coupling (`K = 3‚Äì5` steps)

A one-shot two-stream network already helps, but the LPD literature (R040) shows that *unrolling* the primal/dual
proximals with the forward operator in the loop is what delivers the large CT gains (their Fig./results: +6‚Äì10 dB
over FBP, +2.2 dB over U-Net post-processing, with only ~2.4e5 params and no initial reconstruction). We adopt the
same unrolled skeleton, specialized to our projector so the primal update is a **projection, not a CG solve** (MoDL
/ VN / ADMM-Net, R038/R039/R041, all need an iterative or FFT data-consistency solve because they lack an exact
null projector; we own `P_0`, so ours is closed-form).

State: primal `x_k` (image, extended to `N=5` memory channels as in R040), dual `h_k` (measurement space, `m`
coordinates, `N` memory channels). For `k = 0 ‚Ä¶ K‚àí1`:

```
# dual (measurement-residual) proximal ‚Äî cleans the range / estimates noise
h_{k+1} = Œì_Œ∏^k( h_k,  A x_k^(0),  y )                                   (Eq. HQ14-2)
r_k     = A‚ÄÝ ( dual-to-measurement(h_{k+1}) )        # a range-space correction

# primal proximal ‚Äî writes detail into the null space, applies range correction
x_{k+1} = x_k  +  P_R ¬∑ Œõ_œÜ^k( r_k, x_k )  +  P_0 ¬∑ G_œà( d_A, d_G, x_k )  (Eq. HQ14-3)
```

- `Œì_Œ∏^k` (dual proximal) and `Œõ_œÜ^k` (primal proximal) are per-step learned residual CNNs `Id + convs + PReLU`,
  exactly as in R040; per-iteration weights are **not** shared (R040 finds distinct proximals per step help).
- The forward `A(¬∑)` and adjoint `A·µÄ(¬∑)` are embedded as differentiable layers (already available:
  `A_forward` / `AT_forward` in `ExactRangeNullProjector`; the row-projection path is differentiable).
- The dual stream sees the current measurement residual `A x_k ‚àí y` and learns a **noise-aware** data term:
  where the residual is consistent with the noise statistics it stops pushing, rather than driving `A x_k ‚Üí y`
  hard. This is the Chambolle‚ÄìPock/PDHG dual update with the fidelity `prox` learned (R040), the variational
  data-term weight `Œª_t` learned per step (R039), and the ADMM multiplier learned (R041) ‚Äî all three canonical
  unrolled families collapse to this parameterization.

### 2.2 Soft, noise-aware measurement consistency (the one relaxation)

The exact invariant is deliberately relaxed **only in the range stream**: instead of `A x_hat = y`, HQ-14 targets

```
A x_hat ‚âà y      with      ‚ÄñA x_hat ‚àí y‚Äñ  ‚âà  ‚Äñe‚Äñ  ‚âà  œÉ ‚àöm       (Eq. HQ14-4)
```

so the reconstruction fits `y` to *within the noise floor* rather than exactly ‚Äî the DPS "soft gradient beats hard
projection under noise" observation (R071 / card #70) and the DDRM per-singular-value œÉ-aware blend
(R070 / card #69), specialized to our range/null split. The **null stream is untouched**: `P_0 G_œà` still adds
zero measurement signal by construction (`A P_0 = 0`), so *no hallucination is smuggled into the measured
coordinates*; the relaxation buys only range-space denoising, never fabricated measured content. This preserves
the paper's core stance ‚Äî measured coordinates are never invented ‚Äî while acknowledging the measured coordinates
are now *noisy observations to be denoised*, not ground truth to be pinned.

### 2.3 Training objective

Priors frozen (VQAE, VQGAN, LMMSE covariance). Only `F_œÜ` and the LPD proximals `{Œì_Œ∏^k, Œõ_œÜ^k}` train:

```
L =  Œª_dist ¬∑ ‚Äñx_hat ‚àí x*‚Äñ¬≤                          # distortion (range denoising target)
   + Œª_perc ¬∑ LPIPS(x_hat, x*)                        # perception (R063)
   + Œª_cons ¬∑ ( ‚ÄñA x_hat ‚àí y‚Äñ ‚àí œÉ‚àöm )‚Çä¬≤               # noise-aware consistency band, Eq. HQ14-4
   + Œª_rng  ¬∑ ‚Äñ A ¬∑ P_R F_œÜ(x0,y) ‚àí ( y ‚àí A x* )‚Äñ¬≤    # DDN range-residual = noise, R022 œÜ‚ÇÅ term
```

- The `Œª_rng` term is the Deep-Decomposition bound that ties `F_œÜ` to the *actual noise* `y ‚àí A x* = e`
  (R022: `F` bounded by noise magnitude via `‚ÄñH F ‚àí Œµ‚Äñ`); it is what makes `F_œÜ` a denoiser rather than a free
  range hallucinator. It requires ground-truth `x*` at train time (we have it on STL10), not at test time.
- `Œª_cons` penalizes fitting the range *tighter* than the noise floor (over-fitting `e`) and looser than it
  (under-using the data) ‚Äî a soft band, not a hard equality.
- `G_œà`'s scalar `B` (or the band schedule) is left frozen at the locked operating point so the perception side is
  held fixed and any measured PSNR change is attributable to `F_œÜ`, not to re-tuning the detail dial.

### 2.4 Inference

Feed `y`, compute `x0`/`d_A`/`d_G` as today, run the `K`-step LPD loop (Eqs. HQ14-2/3), emit `x_hat`. **Report both**
`‚ÄñA x_hat ‚àí y‚Äñ / ‚Äñy‚Äñ` (soft-consistency residual, expected `‚âà œÉ‚àöm/‚Äñy‚Äñ`, *not* 2e-15) and the standard
LPIPS/PSNR/RAPSD. A companion **exact-projected variant** `x_hat_exact = audit_flat(x_hat, y)` is also logged: it
re-pins to `y` and thereby *re-injects* `A‚ÄÝ e`, quantifying precisely how much PSNR the noise floor costs ‚Äî this is
the ablation that isolates the value of soft consistency (¬ß5).

## 3. Proposition (range-cleanup PSNR gain is real and bounded)

> **Proposition HQ14-1 (noise floor removed by the range stream).**
> Let `y = A x* + e`, `e ~ (0, œÉ¬≤ I_m)`, with `A` having orthonormal rows in the whitened basis (our operators are
> row-orthonormalized to tolerance `1e-7`, cf. `row_orthonormal_tol` in `get_exact_projector`), so `A A‚ÄÝ = I_m` and
> `‚ÄñA‚ÄÝ e‚Äñ¬≤ = ‚Äñe‚Äñ¬≤`. Then:
>
> (a) **Exact-consistency floor.** *Every* estimator with `A x_hat = y` (including our locked fusion for all `B`
> and all null-space fillers) has range-space error `‚ÄñP_R(x_hat ‚àí x*)‚Äñ¬≤ = ‚ÄñA‚ÄÝ e‚Äñ¬≤ = ‚Äñe‚Äñ¬≤`, independent of the
> filler. Its expected value is `E‚Äñe‚Äñ¬≤ = m œÉ¬≤`, a filler-independent additive PSNR floor.
>
> (b) **Denoising admits strict improvement.** The range-space MMSE estimate `E[P_R x* | y]` (LMMSE shrinkage of the
> measured modes) has strictly smaller expected range error than `A‚ÄÝ y` whenever `œÉ¬≤ > 0` and the prior on the
> measured coordinates is non-degenerate (finite prior variance `œÑ¬≤`): the per-mode shrinkage `œÑ¬≤/(œÑ¬≤+œÉ¬≤) < 1`
> gives range MSE `m ¬∑ œÉ¬≤œÑ¬≤/(œÉ¬≤+œÑ¬≤) < m œÉ¬≤`. A range stream `F_œÜ` with enough capacity to realize this shrinkage
> therefore lowers total distortion by exactly the gap `m œÉ¬≤ ¬∑ (1 ‚àí œÑ¬≤/(œÉ¬≤+œÑ¬≤)) = m œÉ‚Å¥/(œÉ¬≤+œÑ¬≤) > 0`.
>
> (c) **Null stream is unaffected.** Because `A P_0 = 0`, `F_œÜ` (range) and `G_œà` (null) contribute orthogonal
> error components; the range cleanup in (b) leaves the null-space LPIPS budget of the frozen fusion unchanged.
> Hence at fixed perception (fixed `G_œà`, `B`) the two-stream estimator's distortion is `‚â§` the exact fusion's,
> with strict inequality iff `œÉ¬≤ > 0`.

*Proof sketch.* (a) is immediate from `P_R x_hat = A‚ÄÝ A x_hat = A‚ÄÝ y` and `A‚ÄÝ y ‚àí P_R x* = A‚ÄÝ e`. (b) is the scalar
Wiener/LMMSE result applied per measured mode (independence follows from row-orthonormality + isotropic `e`);
it coincides with the posterior-mean range component that the P‚ÄìD theorem (R061, Thm. 3) identifies as
distortion-optimal. (c) is orthogonality of `P_R`, `P_0`. ‚àé

**Reading.** The proposition says the *maximum* recoverable PSNR from HQ-14 over the exact fusion is the noise
floor `m œÉ‚Å¥/(œÉ¬≤+œÑ¬≤)` ‚Äî large when `œÉ` is comparable to the measured-mode signal `œÑ` (low-SNR GI), vanishing as
`œÉ ‚Üí 0` (our current noiseless testbed). This is the honest ceiling: HQ-14 is a *noise-regime* method, and its
gain is exactly the noise the exact invariant is forced to keep.

## 4. Relation to prior art (grounded, positioned)

| Card | What we take | What we change |
|------|--------------|----------------|
| R022 Deep Decomposition (#15) | `A(y)=A‚ÄÝy + P_R F + P_0 G` split; `F` bounded by noise via `‚ÄñHF‚àíŒµ‚Äñ` | matrix-free exact `P_0` (no `H·µÄ` denoiser needed); `G` = frozen VQAE+VQGAN, not learned-vs-GT; **we don't claim the measurement certifies `G`** ‚Äî only `F` is supervised in the range |

| R039 Variational Network (#28) | per-step learned data-term weight `Œª_t` (fidelity ‚Üî prior) | `Œª_t` becomes a *noise-aware* soft band (Eq. HQ14-4), not a hard fit |
| R041 Deep ADMM-Net (#30) | learned multiplier/shrinkage, init to replicate a classical baseline | init so step-0 = today's exact fusion, so HQ-14 can only match-or-beat at `œÉ‚Üí0` |
| R061 P‚ÄìD tradeoff (#50) | posterior-mean range = distortion-optimal under noise; `Œª`-as-Lagrange dial | our range cleanup realizes exactly the distortion-optimal shrinkage the exact invariant forbids |
| R070 DDRM / R071 DPS (#69/#70) | œÉ-aware measured-mode blend; soft gradient beats hard projection under noise | applied only to the **range** stream; null stream stays measurement-invisible |

**Novelty against the closest relative (R022).** Deep Decomposition learns *both* streams supervised against
ground truth to raise PSNR and treats the null content as trustworthy output. Our program's thesis is the
opposite: null content is *unverifiable* and must be *governed*, not certified. HQ-14 keeps that thesis intact ‚Äî
`G_œà` (null) remains the frozen, governed, uncertifiable detail prior; only `F_œÜ` (range) is supervised, and it
operates exclusively on measured coordinates the measurement *does* constrain. So HQ-14 is R022's architecture
running under our epistemics: **denoise where the data lives, govern where it doesn't.** That framing is, to our
knowledge, not in the decomposition-learning literature and is the paper-worthy contribution here.

## 5. Ablation / experiment design

**Testbed.** A *noisy* 5% operator: take the locked structured 5% `A`, add Gaussian read-noise `e ~ N(0, œÉ¬≤I_m)`
to `y` at three SNRs (`œÉ` set so measured-mode SNR ‚âà 30 / 20 / 10 dB; report `œÉ` in whitened units, no invented
numbers). Also run the clean `œÉ=0` case as the control that *must* show no HQ-14 gain. STL10 64√ó64, seeds 0/1/2,
same val‚Üídev multi-seed gate firewall as every other arm (`cmd_canary` / `cmd_gate`). Extends the operator plumbing
in `gan_high_quality_gi.py` (add `read_noise_sigma` to the operator config; `y = A x + œÉ Œµ`).

**Arms (all at each `œÉ`):**
1. `exact_fusion` ‚Äî locked null-only fusion forced to fit noisy `y` (baseline; range error = `A‚ÄÝ e`).
2. `lmmse_anchor_exact` ‚Äî current `EmpiricalLMMSE.anchor()` with its `audit_flat` re-pin (shows the anchor also
   carries `A‚ÄÝ e`).
3. `F_only` ‚Äî two-stream **one-shot** (`K=1`, Eq. HQ14-1), `G_œà` frozen at locked `B`; isolates pure range denoising.
4. `LPD_K3`, `LPD_K5` ‚Äî full unrolled two-stream (Eqs. HQ14-2/3).
5. `LPD_K5 + hard reproject` ‚Äî `audit_flat(x_hat, y)` re-pins to `y`; **must lose PSNR** vs arm 4, quantifying the
   noise floor (this is the key ablation proving soft-consistency is doing the work, not extra capacity).

**Metrics.** PSNR, LPIPS (R063), RAPSD; plus `‚ÄñA x_hat ‚àí y‚Äñ/‚Äñy‚Äñ` (expected `‚âà œÉ‚àöm/‚Äñy‚Äñ` for arms 3‚Äì4, `‚âà2e-15` for
arms 1,2,5). Report the P‚ÄìD plane (LPIPS vs PSNR) per `œÉ`.

**Pre-registered success criterion.**
- **Noisy (`œÉ>0`):** `LPD_K5` achieves **higher PSNR at equal-or-better LPIPS** than `exact_fusion` on ‚â•2/3 seeds
  through the locked gate, and `LPD_K5 + hard reproject` is strictly worse than `LPD_K5` (confirming the gain is
  range denoising, per Prop. HQ14-1(b)).
- **Clean (`œÉ=0`, control):** HQ-14 shows **no significant PSNR/LPIPS change** vs `exact_fusion` (it must not
  "improve" a noiseless case ‚Äî if it does, `F_œÜ` is hallucinating in the range and the `Œª_rng`/`Œª_cons` terms are
  mis-set). This control is as important as the positive result: it protects the whole program's exact-consistency
  claim on the clean testbed.

**Ablation knobs.** (i) `K ‚àà {1,3,5}` ‚Äî does unrolling help beyond one-shot? (R040 predicts yes under noise).
(ii) drop `Œª_rng` ‚Äî does `F_œÜ` drift into free range hallucination (consistency band violated, clean-control fails)?
(iii) share vs per-step proximals (R040 predicts per-step wins). (iv) `F_œÜ` capacity sweep vs the Prop. HQ14-1(b)
ceiling `m œÉ‚Å¥/(œÉ¬≤+œÑ¬≤)` ‚Äî verify measured gain saturates at the theoretical floor, not beyond it (a gain *beyond*
the floor is a red flag for null leakage).

## 6. Honest gain assessment vs the current fusion

- **In the current locked (noiseless) regime: gain is zero by design, and that is the correct outcome.** Prop.
  HQ14-1 makes the noise floor `‚Üí 0` as `œÉ‚Üí0`, so there is nothing for `F_œÜ` to remove; the exact fusion is
  already distortion-optimal in the range and quality-optimal via `G_œà` in the null. HQ-14 must *tie* the clean
  control, not beat it.
- **In the noisy / real-GI regime: gain can be large, and it is the noise the exact invariant is forced to keep.**
  The recoverable PSNR is exactly `m œÉ‚Å¥/(œÉ¬≤+œÑ¬≤)` (Prop. HQ14-1(b)) ‚Äî sizable at low measured-mode SNR (the
  operating point real bucket-detector GI actually lives in), negligible at high SNR. R040/R022 report multi-dB
  range-cleanup gains in the analogous CT/MRI noisy settings, which sets a plausible (not promised) expectation.
- **What HQ-14 is really contributing to the paper is not a number but a *regime*:** it extends the null-space
  fusion story from "clean-`y`, exact consistency, quality bought in `Null(A)`" to "noisy-`y`, soft
  noise-aware consistency, quality bought in `Null(A)` *and* fidelity recovered by denoising `Range(A·µÄ)`" ‚Äî while
  keeping the epistemic firewall (null = governed/uncertifiable, range = data-constrained/denoisable) intact. That
  is a robustness section, not a headline LPIPS win, and it should be framed as such. Marginal on the clean bench;
  potentially decisive under noise.

**Bottom line: propose it as the dedicated noise-robustness arm; do not fold it into the clean-`y` Pareto or the
`‚àí32.6% LPIPS` claim.** Its ceiling on the current testbed is zero; its value materializes only when a noisy
operator is instantiated (a one-line config change to `gan_high_quality_gi.py`), at which point it is the only arm
in the deck that can move PSNR at all.
