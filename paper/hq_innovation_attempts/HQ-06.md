# HQ-06 — Null-Space MoDL: an unrolled data-consistency-free refiner that replaces the scalar dial `B`

*Artifact for the high-quality-GI-reconstruction line ("raise the reconstruction-quality ceiling of exact null-space fusion"). This point replaces the single global scalar `B` in the locked fusion with a `K`-stage weight-shared unrolled refiner that lives **entirely inside `Null(A)`**, so measurement consistency is free-by-construction and the current locked Pareto point is recovered as the stage-0 identity. Every method statement is grounded in the reading cards named inline; no numbers are invented — the one reported operating point (`−32.6%` LPIPS at bounded PSNR) is the repository's locked result, and it is used only as the initialization target and the success bar.*

**Invariant respected (unchanged from the locked line).** All edits live inside the exact null projector: `x̂ = x0 + P0(·)`, so `A x̂ = y` to the numerical floor by construction because `A P0 = 0`. HQ-06 does **not** relax this. Quality is bought only by *what content the unrolled net writes into `Null(A)`*; the range component is never touched, so this is the clean-`y` quality arm, distinct from the soft-consistency noisy-`y` arm (HQ-14).

---

## 1. The gap this attacks

The locked detail-fusion writes a single null-space increment governed by one global scalar:

> `x̂_B = x0 + P0( d_A + B (d_G − d_A) )`,  `B ∈ [0,1]`

(this is exactly `fuse(("scalar", B), x0f, d_A, d_G, …)` in `vqgan_detail_fusion.py`, which computes `d_F = null_project_flat(d_A + B·(d_G − d_A))` then `xhat = audit_flat(x0f + d_F, y)`). `d_A` (VQAE structure) and `d_G` (VQGAN detail) are already null-projected residuals from `prep_residuals`.

`B` is **one number for the whole image**. It cannot say "trust VQGAN detail in this textured patch but stay near the VQAE structure in that smooth region," and — more importantly for the *ceiling* — it can only ever return a point on the chord between two frozen residuals `d_A` and `d_G`. It cannot **synthesize** null content that neither prior produced. HQ-06's thesis: an unrolled, spatially-adaptive, iteratively-refining denoiser confined to `Null(A)` can both (i) vary the perception–distortion mix per region and per stage, and (ii) generate detail off the `d_A`–`d_G` chord, pushing LPIPS below the locked operating point without paying PSNR.

## 2. Method

### 2.1 Grounding: why "MoDL without the CG block"

The classical unrolled model-based reconstructor MoDL (R038) alternates a weight-shared CNN denoiser with a **data-consistency (DC) subproblem** it must solve numerically: `x_{n+1} = (AᴴA + λI)⁻¹(Aᴴb + λ z_n)`, `z_n = D_w(x_n)`, with the DC step run by an in-network conjugate-gradient (CG) solve because a general forward operator has *no analytic inverse* (R038 method-core). The CG block exists **only** to project the denoiser output back toward measurement consistency; MoDL pays for it because it lacks an exact null projector.

We own one. `ExactRangeNullProjector` in `src/projections.py` supplies `null_project_flat` (`P0 = I − AᵀG⁻¹A`) and `data_anchor_flat`/`audit_flat` that pin the range to `y` exactly. So the entire MoDL DC subproblem collapses to a single closed-form re-projection: **there is no CG block, no `λ`, no normal-equation solve** — data consistency is achieved by one matrix-free application of `P0` per stage. This is precisely the structure the null-space-network theory certifies: Schwab–Antholzer–Haltmeier's Deep Null-Space Learning (R021) proves that any operator of the form `L = Id + P_ker(A)·N` satisfies `A L x = A x` for all `x` (exact consistency for any Lipschitz `N`), and is a convergent regularization; our unrolled net *is* such an `L` iterated `K` times. Deep Decomposition Learning (R022) gives the same range/null split we use (`P_r = HᵀH`, `P_0 = I − HᵀH`), and its ablation is the empirical mandate for spending capacity in the null space: using **both** projected streams beat null-only, but crucially null-only itself lifted quality substantially over a raw residual net (R022 key-results). We stay null-only for clean `y` (the range is already exact); the range stream is deferred to HQ-14.

The per-stage denoiser is patterned on ISTA-Net (R042): its per-phase learned prox `x = F̃(soft(F(r),θ))` around a **fixed** gradient step that "touches only the range of Φ, while the learned prox is free to invent null-space content" (R042 innovation-spark). HQ-06 keeps ISTA-Net's per-phase learned nonlinearity but hard-confines the invented content to `Null(A)` via `P0` instead of ISTA-Net's soft fidelity gradient.

### 2.2 The unrolled null-space refiner

Let `x0f` be the flattened LMMSE anchor and `d_A, d_G` the two null-projected residuals (all as returned by `prep_residuals`, float64). Define a **stack of `K` weight-shared stages** whose state is a candidate reconstruction `x_k`. Each stage applies a small denoiser `D_w` and re-projects its *increment* into the null space:

> **Stage update (k = 0 … K−1):**
> `z_k   = D_w( x_k ; d_A, d_G )`          (learned denoiser, two residuals as side channels)
> `x_{k+1} = x0f + P0( z_k − x0f )`         (`= x0f + null_project_flat(z_k − x0f)`)
> **Final:** `x̂ = audit_flat( x_K , y )`      (one float-drift audit against `y`)

Because `x_{k+1} − x0f = P0(z_k − x0f) ∈ Null(A)`, we have `A x_{k+1} = A x0f = y` **at every stage** (the anchor is measurement-consistent), so the invariant `A x̂ = y` holds at every intermediate `x_k`, not just at the end. `audit_flat` only cleans float64 round-off (`relmeaserr → ~2e-15`); it is not doing the consistency work — `P0` is.

**Denoiser `D_w`.** A tiny residual CNN with weight-sharing across all `K` stages (MoDL's parameter-economy trick, R038 key-results: weight-sharing gives ~10× fewer parameters and is insensitive to training-set size beyond ~100 images — ideal for the small STL10 64×64 testbed). Input is a **5-channel** image-space tensor: the current estimate `x_k` (1ch, or 3 if RGB), plus the two frozen residuals `d_A`, `d_G` reshaped to image space as *side information* so the net can learn a spatially- and iteration-varying blend **and** additive detail. Output is a residual: `D_w(x_k) = x_k + N_w([x_k, d_A_img, d_G_img])`. Target size ≈ 100k parameters (weight-shared), K = 3–6 stages.

### 2.3 Stage-0 identity initialization (the "can only improve" guarantee)

Initialize the network so that **stage 0 exactly reproduces the locked operating point** `x̂_{B*}`. Two equivalent ways:

1. *Warm-start the state:* set `x_0 = x̂_{B*} = x0f + P0(d_A + B*·(d_G − d_A))` at the locked `B*` (the balanced-gate winner). Then set `D_w = Id` (zero the residual branch `N_w` at init) so `x_1 = x0f + P0(x_0 − x0f) = x_0` (idempotence of `P0` on a vector already in the anchor+null decomposition). Stage-0 output = the locked Pareto point exactly.
2. *Fold `B*` into the first conv bias* so the untrained net emits `d_A + B*·(d_G − d_A)` as its increment.

Either way, at initialization HQ-06 **is** the locked fusion; training can only move off that point if it lowers the loss. This mirrors the Deep-Decomposition / ISTA-Net practice of initializing the unrolled net to replicate a classical baseline and then optimizing to beat it (R042 method-core: `Q_init` least-squares init; ADMM-Net-style "initialize to replicate the l1 baseline, then optimize").

### 2.4 Training recipe

- **Frozen:** the LMMSE anchor `x0` (`hq.EmpiricalLMMSE`), both refiners that produce `d_A`, `d_G`, the operator `A`, and `P0`. **No prior/operator retraining.** Only `D_w` (one shared block) trains.
- **Trainable params:** `D_w` weights + a single learned per-stage step scalar `η_k` (optional, ISTA-Net-style learned step; default tied to 1). ≈100k total.
- **Data:** STL10 train split, 64×64, cached `(x0f, d_A, d_G, y, truth)` packs from `prep_residuals` — identical caching path the canary already uses (`load_pack`). Because the packs are precomputed, a training step is: run `K` stages (each = one `P0` application + one tiny CNN forward), compute loss, backprop. `P0` is matrix-free and differentiable (`row_project_flat`/`null_project_flat` are plain tensor ops), so gradients flow through every stage.
- **Loss:** `L = LPIPS(x̂, truth) + γ·MSE(x̂, truth)`, with `γ` chosen so the MSE term prevents PSNR regression below the anchor (the same PSNR-guard the `select_operating_point("balanced")` gate enforces). No consistency term is needed — `A x̂ = y` is analytic, exactly as Deep Null-Space Learning (R021) guarantees for `L = Id + P_ker N`.
- **Optimizer / schedule:** Adam, ~1 GPU-hour on the 64×64 testbed (small net, cached residuals, K≤6 forward passes). Train on **dev** only; **never** touch the locked val→dev gate splits during training.
- **Inference:** feed-forward, `K` stages, then one `audit_flat`. Deterministic; no sampling.

### 2.5 Where it plugs into the code

Add a spec `("modql", ckpt)` handled in `fuse()` (or a sibling `fuse_unrolled`) that, instead of the scalar/band branch, runs the `K`-stage loop above and returns `measurement.unflatten_img(audit_flat(x_K, y))`. Everything downstream — `fast_means`, `select_operating_point`, `cmd_canary`/`cmd_gate` — is reused verbatim, so HQ-06 is scored through the identical locked firewall as the `scalar`/`band` arms.

## 3. Proposition (exact consistency and non-regression of the unrolled refiner)

**Setup.** Let `A ∈ ℝ^{m×n}`, `P0 = I − AᵀG⁻¹A` the exact null projector (`A P0 = 0`), and `x0f` a measurement-consistent anchor (`A x0f = y`, true for `data_anchor_flat(y)` and for the LMMSE anchor as used). Let `D_w` be any Lipschitz denoiser and define the `K`-stage recursion `x_{k+1} = x0f + P0(D_w(x_k) − x0f)`, with output `x̂ = x_K`.

**Claim.**
1. **Exact consistency at every stage.** For all `k`, `A x_k = y`. In particular `A x̂ = y` regardless of `D_w`, `K`, or the training state.
2. **Stage-0 non-regression.** If `D_w` is initialized so that `x_1 = x̂_{B*}` (§2.3), then the untrained HQ-06 equals the locked fusion output, and any trained state that decreases the training loss produces a reconstruction with loss ≤ that of `x̂_{B*}` on the training distribution.

**Proof.**
1. Induction on `k`. Base: `A x_0 = y` (anchor consistency, or `x_0 = x̂_{B*}` which satisfies `A x̂_{B*} = A x0f = y` since its edit is in `Null(A)`). Step: `A x_{k+1} = A x0f + A P0(D_w(x_k) − x0f) = y + 0 = y`, using `A P0 = 0`. This is exactly the Deep-Null-Space identity `A L x = A x` (R021) applied stage-wise, and the same geometry the locked `fuse` uses for its single increment. ∎(1)
2. At init `x_1 = x̂_{B*}` by construction, so the loss equals the locked point's loss. Training minimizes the same loss over `D_w`; a minimizer cannot have larger loss than the init, so on the training distribution loss(trained) ≤ loss(`x̂_{B*}`). ∎(2)

**Honesty caveat (load-bearing).** Claim 2 is a statement about the **training loss on the training distribution**, not a guarantee on the held-out gate. Non-regression at the operating point is only guaranteed *up to* the val→dev generalization gap; the locked multi-seed gate is what actually certifies improvement, and it can reject HQ-06 if the extra capacity overfits dev. Claim 1 (exact consistency) is unconditional. Crucially, exact consistency does **not** certify the *correctness* of the invented null content — R022/R021 fill the null space to improve quality, but the measurement never vouches for it; that is the whole-program stance and it is unchanged here.

## 4. Ablation / experiment design

All runs go through the existing `cmd_canary` (dev) and `cmd_gate` (locked val→dev firewall) at 2/5/10% on structured DCT+Hadamard+random operators, seeds 0/1/2, so results are directly comparable to the locked `scalar_B` Pareto.

**A. Primary head-to-head (does it beat the locked point).**
Arms scored at 5%: (i) `scalar_B*` locked baseline; (ii) HQ-06 `K=3`; (iii) HQ-06 `K=6`. Select each via `select_operating_point("balanced")` on the gate. **Success = LPIPS below the locked `−32.6%` at equal-or-better PSNR on ≥2/3 seeds.** Report the full LPIPS-vs-PSNR frontier, not a single point.

**B. Depth ablation (`K`).** `K ∈ {1, 3, 6}` (K=1 with `D_w=Id` reproduces the locked point — the identity control). Plot LPIPS/PSNR vs `K`; expect saturation (MoDL saturates ~10 iters, R038; ISTA-Net error saturates after a handful of phases, R042). This shows whether iteration (synthesis off the chord) or just spatial adaptivity is responsible for any gain.

**C. Side-channel ablation (is it just re-learning `B`?).** Three `D_w` inputs: (c1) `x_k` only — no `d_A`/`d_G` side channels; (c2) `[x_k, d_A, d_G]` (full); (c3) `[x_k, d_A, d_G]` but with the residual branch **constrained** to output a per-pixel convex blend `β(x)·(d_G − d_A)` (this is the HQ-05 `β(x)` special case — a reweighting-only net). If (c2) ≫ (c3), the gain comes from **synthesis** (content off the chord), which is the entire ceiling argument; if (c2) ≈ (c3), HQ-06 has degenerated into a spatial `β` gate and the honest conclusion is a marginal win (see §5).

**D. Consistency audit.** Log `relmeaserr` at every stage for a batch; confirm `≤ 1e-5` (target `~2e-15` after `audit_flat`) for all `K`, all seeds — the exact-consistency figure.

**E. Convergence trace.** Log `‖x_{k+1} − x_k‖` across stages for the convergence figure (parallels HQ-07's RED trace), demonstrating the unrolled refiner reaches a stable estimate rather than drifting.

**Compute.** ≈1 GPU-hour train + inference-time eval; cached residuals make each stage cheap.

## 5. Honest assessment of expected gain — **likely marginal-to-moderate, with a real chance of degenerating to `≈ B`**

This is the crux the deck flags for HQ-06, and I keep it front-and-center rather than overselling.

**The optimistic case (moderate gain).** Unlike a per-image scalar (HQ-08) or a static radial band, HQ-06's denoiser can (i) vary the mix per region and per stage and (ii) — the only mechanism that can raise the *ceiling* rather than just re-slice the chord — **synthesize additive null content** `N_w([x_k,d_A,d_G])` that is neither `d_A` nor `d_G`. That additive term is what could push LPIPS below `−32.6%` at fixed PSNR. Deep Decomposition (R022) is the precedent that spending learned capacity in the null space lifts quality, and MoDL/ISTA-Net (R038/R042) show unrolling with weight-sharing is data-efficient enough for a ~2000-image STL10 testbed.

**The pessimistic case (marginal gain — and it is the more likely one).** The deck's own risk note is correct and I do not soften it: a *prior learned gate already underperformed* on this program, and HQ-05 (`β(x)`) / HQ-06 are both "spatially-adaptive learned blends of the same two residuals." The dominant failure mode is that `D_w` learns `≈` the constant-`B` map plus imperceptible noise, because:
- The reachable improvement is bounded by how much **region-to-region** (not image-to-image) prior-trust variation actually helps — and if `d_A` and `d_G` largely agree in the null space (small `‖d_G − d_A‖`), there is little for any adaptive blend to exploit.
- LPIPS-driven synthesis in `Null(A)` is *measurement-invisible by construction*, so the net can inject perceptually-scored-but-wrong texture; the PSNR guard (`γ·MSE`) then pulls it back toward the chord, collapsing the extra degrees of freedom. This is the same tension that makes null-only synthesis intrinsically capped.

**Why HQ-06 is still the right member to try over HQ-05.** Between the two spatially-adaptive learned blends, MoDL's iterative denoiser is the higher-ceiling one because it can **synthesize content, not just reweight** — ablation **C** is designed precisely to test whether that extra capability delivers or degenerates. If ablation C shows (c2) ≈ (c3), the honest paper claim is "a spatial/iterative refiner matches but does not beat the tuned scalar dial, confirming the scalar `B` is near-optimal for two-residual null fusion" — a legitimate negative/tightness result for the governance story, not a headline quality win.

**Bottom line.** Expected LPIPS gain over the current fusion is **small in the likely case, moderate only if additive null synthesis (ablation C, arm c2) meaningfully beats pure reweighting (c3).** The value of running it is twofold regardless of outcome: (a) a possible modest ceiling lift at ~1 GPU-hour and zero prior retraining; (b) a clean tightness result bounding how much *any* learned adaptive blend of `d_A`/`d_G` can extract, which directly supports the "single governed scalar is near-optimal" narrative. HQ-06 is a near-term win to attempt, not a moonshot, and it must not be written up as a guaranteed improvement.
